### 类路径

在 Spring（尤其是 **Spring Boot**）项目中，`src/main/resources` 目录下的路径通常被称为：

> **Classpath 路径（classpath path）** 或 **类路径资源（classpath resources）**

------

#### 📁 具体说明

##### 1. **`resources` 目录的作用**

- `src/main/resources` 是 Maven/Gradle 项目的标准目录。
- 构建时，该目录下的所有内容会被**复制到 classpath 根目录**下（即最终打包进 JAR/WAR 的 `BOOT-INF/classes/` 或 `WEB-INF/classes/` 中）。

##### 2. **如何引用这些资源？**

在代码中通过以下方式访问：

```java
// 方式1：使用 ClassPathResource（推荐）
Resource resource = new ClassPathResource("application.yml");

// 方式2：使用 ClassLoader
InputStream is = getClass().getClassLoader().getResourceAsStream("static/index.html");

// 方式3：在 Spring 配置中直接写路径（如 @Value、@PropertySource）
@Value("classpath:templates/email.html")
private Resource emailTemplate;
```

> ✅ 所有路径都是**相对于 classpath 根目录**的，**不需要加 `resources/` 前缀**！

例如：

- 文件物理位置：`src/main/resources/static/logo.png`
- Classpath 路径：`static/logo.png`
- 访问方式：`new ClassPathResource("static/logo.png")`

------

#### 🧭 常见子目录及其用途（Spring Boot 默认约定）

| Classpath 路径                                  | 用途                                                         |
| ----------------------------------------------- | ------------------------------------------------------------ |
| `/static/`                                      | 静态资源（HTML, CSS, JS, 图片等），可直接通过 HTTP 访问      |
| `/public/`                                      | 同上，优先级低于 `/static`                                   |
| `/templates/`                                   | 模板文件（Thymeleaf, FreeMarker 等），**不能直接 HTTP 访问** |
| `/application.properties` 或 `/application.yml` | 主配置文件                                                   |
| `/messages/`                                    | 国际化资源文件（如 `messages_en.properties`）                |

------

#### 🔑 关键概念总结

| 术语                          | 含义                                                   |
| ----------------------------- | ------------------------------------------------------ |
| **Classpath**                 | JVM 加载类和资源的搜索路径，`resources` 内容会进入这里 |
| **Classpath 资源**            | 位于 classpath 下的文件，可通过 `classpath:` 协议访问  |
| **资源路径（Resource Path）** | 指从 classpath 根开始的相对路径，如 `static/app.js`    |

------

#### ❗ 注意事项

- 在代码中**永远不要写 `resources/xxx`**，因为构建后 `resources` 目录本身不存在于 classpath 中。

- 使用 `Resource` 接口或 `ResourceLoader` 是 Spring 推荐的资源访问方式，支持 classpath、file、URL 等多种协议。

- 在 

  ```
  application.properties
  ```

   中引用资源时，也用 

  ```
  classpath:
  ```

   前缀：

  ```properties
  my.custom.file=classpath:config/myfile.txt
  ```

------

#### ✅ 示例

假设项目结构如下：

```
src/
 └── main/
     └── resources/
         ├── application.yml
         ├── static/
         │   └── index.html
         └── templates/
             └── welcome.html
```

则：

- `index.html` 的 classpath 路径是：`static/index.html`
- 浏览器访问：`http://localhost:8080/index.html`（由 Spring Boot 静态资源处理）
- 在 Java 中读取：`new ClassPathResource("static/index.html")`
- `welcome.html` 的 classpath 路径是：`templates/welcome.html`（但不能通过浏览器直接访问）
