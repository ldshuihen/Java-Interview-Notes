## 1. 进程和线程的区别是什么？
- 进程是**资源分配的基本单位**，线程是**CPU 调度的基本单位**。
- 进程有独立的地址空间、文件描述符、内存等，线程共享所属进程的资源。
- 进程切换开销大，线程切换开销小。
- 一个进程可以包含多个线程，进程崩溃一般不影响其他进程；线程崩溃可能导致整个进程崩溃。
- 进程间通信复杂，线程间通信简单（共享内存即可）。

## 2. 进程有哪些状态？状态如何转换？
经典三态：
- **就绪**：已获得除 CPU 外的所有资源，等待调度。
- **运行**：占用 CPU 执行。
- **阻塞**：等待某事件（IO、信号、锁），暂时无法运行。

转换：
- 就绪 → 运行：被调度选中。
- 运行 → 就绪：时间片用完、被抢占。
- 运行 → 阻塞：等待资源/IO。
- 阻塞 → 就绪：等待的事件完成。

扩展五态：新建、就绪、运行、阻塞、终止。

## 3. 什么是上下文切换？开销主要在哪？
当前线程/进程的 CPU 上下文（寄存器、程序计数器、页表等）保存，再加载下一个的上下文，这个过程叫上下文切换。

开销主要在：
- 保存/恢复寄存器
- 刷新 TLB、缓存失效
- 内核态切换开销
- 调度算法本身开销

频繁切换会导致 CPU 利用率下降，也就是**调度消耗大于实际执行**。

## 4. 线程的实现方式：用户级线程 vs 内核级线程
- **用户级线程**：内核不知道，由用户态库管理。切换快，但一个阻塞会导致整个进程阻塞，无法多核并行。
- **内核级线程**：内核管理、调度。可多核并行，一个线程阻塞不影响其他，但切换开销更大。
- 现代 OS 多用**一对一模型**（Linux/NPTL、Windows），Java 线程基本都映射到内核线程。

## 5. 什么是协程？和线程的区别？
协程是**用户态轻量级执行单元**，由用户态调度，不陷入内核。

区别：
- 线程是内核调度，协程是用户态调度。
- 协程切换几乎无开销，线程切换开销大。
- 线程抢占式调度，协程协作式调度（主动让出）。
- 一个线程可以跑成千上万协程。

## 6. 什么是守护线程？
- 运行在后台，为其他线程服务的线程。
- 当所有**非守护线程**结束时，JVM 会直接退出，不管守护线程是否执行完。
- 比如 GC 线程、编译线程都是守护线程。

## 7. 进程间通信（IPC）方式有哪些？
- 管道/匿名管道
- 命名管道（FIFO）
- 消息队列
- 共享内存（最快）
- 信号量
- 信号
- 套接字（Socket）

## 8. 线程间同步方式有哪些？
- 互斥锁（Mutex）
- 自旋锁
- 读写锁
- 信号量
- 条件变量
- 屏障（Barrier）
Java 层面对应：synchronized、Lock、Semaphore、CountDownLatch、CyclicBarrier 等。

## 9. 什么是死锁？死锁的四个必要条件？
多个进程/线程互相持有对方需要的资源，都不释放，导致永久阻塞。

四个必要条件：
1. **互斥**：资源同一时间只能被一个线程占有
2. **请求与保持**：持有资源并继续请求其他资源
3. **不可剥夺**：资源只能主动释放，不能被强行抢占
4. **循环等待**：形成资源请求环路

四个条件**同时成立**才会死锁。

## 10. 如何避免/预防/解除死锁？
- **破坏互斥**：尽量使用共享资源（不太常用）
- **破坏请求与保持**：一次性申请所有资源
- **破坏不可剥夺**：允许抢占资源
- **破坏循环等待**：资源统一编号，按顺序申请

避免：银行家算法（动态检测）
检测与解除：定时检测环，杀死进程/线程回滚。

## 11. 什么是饥饿、活锁？和死锁区别？
- **饥饿**：某线程长期得不到资源，一直无法运行。
- **活锁**：线程不断重试、状态在改变，但业务没有进展。
- **死锁**：互相阻塞，状态不变，完全不推进。

区别：
- 死锁：都不动
- 活锁：都在动，但没进展
- 饥饿：有人一直动，有人一直不动

## 1. 什么是虚拟内存？作用是什么？
虚拟内存是操作系统给每个进程提供的**独立、连续的逻辑地址空间**，与物理内存不直接对应，通过 MMU + 页表映射。

作用：
- 扩大地址空间，让进程可用内存大于物理内存
- 进程间地址空间隔离，更安全
- 实现内存共享、按需分配、换入换出
- 简化内存管理，程序不用关心物理地址

## 2. 分页与分段的区别？
- **分页**：按**固定大小页**划分，对用户透明，减少外部碎片，但有内部碎片，以页为单位管理。
- **分段**：按**逻辑模块**（代码段、数据段、栈段）划分，大小不固定，便于共享和保护，但易产生外部碎片。

区别点：
- 单位：固定页 vs 可变段
- 地址：一维线性地址 vs 二维（段号+偏移）
- 碎片：内部碎片 vs 外部碎片
- 共享保护：分段更方便，分页较弱

## 3. 什么是缺页中断？
进程访问的虚拟页**不在物理内存中**时，CPU 触发缺页中断。
操作系统会：
- 阻塞当前进程
- 从磁盘 swap 区加载页面到内存
- 更新页表
- 重新执行这条指令

## 4. 页面置换算法有哪些？
- OPT 最优置换（理论）
- FIFO 先进先出
- LRU 最近最少使用（面试最常考）
- LFU 最少使用
- Clock 时钟算法（近似 LRU）

## 5. 什么是内存碎片？内部碎片 vs 外部碎片？
内存碎片是**未被有效利用的空闲内存空间**。

- **内部碎片**：分配给进程的内存 > 实际需要，浪费在分配单元内部
  例：分页、伙伴系统
- **外部碎片**：空闲内存总量足够，但不连续，无法分配给大块需求
  例：分段、早期 malloc 机制

## 6. 堆和栈的区别？
- 栈：由系统自动管理，函数调用开辟释放，连续空间，速度快，线程私有
- 堆：程序员/GC 管理，动态分配，不连续，速度慢，进程内共享
- 生长方向：栈通常向下，堆向上
- 碎片：栈无碎片，堆易产生碎片
- 大小：栈较小固定，堆较大灵活

## 7. 什么是缓冲区溢出？
向固定大小缓冲区写入**超出长度的数据**，覆盖栈上的返回地址、栈帧等，导致程序崩溃或被恶意控制。
常见于 C/C++，Java 因数组越界检查基本不会出现。

## 8. Linux 内存分配：brk、mmap 区别
- brk：移动堆顶指针，分配小块内存，连续，释放后不会立即还给 OS，容易产生内存碎片
- mmap：在文件映射区匿名映射，分配大块内存，独立空间，释放后可立即归还 OS
- 一般阈值（如 128K）：小于用 brk，大于用 mmap

## 9. 大内存页是什么？有什么用？
把标准页（如 4K）换成更大的页（如 2M、1G）。

好处：
- 减少页表项，降低 TLB 失效
- 减少缺页中断，提升性能
- 适合高并发、大数据、JVM 堆场景

# 三、CPU 调度
1. 操作系统调度算法有哪些？

3. 什么是IO密集型/CPU密集型任务？调度策略区别？

## 1. 操作系统调度算法有哪些？
- 先来先服务 FCFS
- 短作业优先 SJF / 短进程优先 SPF
- 优先级调度
- 时间片轮转 RR
- 多级反馈队列调度
- 高响应比优先 HRRN
- 公平共享调度

## 2. FCFS、SJF、优先级调度、时间片轮转、多级反馈队列原理
- **FCFS 先来先服务**
  按照进程到达顺序依次执行，不抢占，简单但不利于短作业、IO 密集型任务。

- **SJF 短作业优先**
  选择预计运行时间最短的进程先执行，平均等待时间最短，但可能导致长作业饥饿。

- **优先级调度**
  按优先级高低选择进程，可分为抢占式与非抢占式；可能出现低优先级进程饥饿。

- **时间片轮转 RR**
  进程按 FCFS 排队，每个进程执行一个固定时间片，时间片到则切换回队列末尾；响应快，适合分时系统。

- **多级反馈队列**
  设置多个优先级队列，优先级越高时间片越短；
  新进程进入最高优先级队列，用完时间片则降级；
  IO 密集型容易保持高优先级，CPU 密集型会逐步降级；
  综合性能好，Linux/Windows 常用。

## 3. 什么是 IO 密集型 / CPU 密集型任务？调度策略区别？
- **CPU 密集型**
  大量计算，CPU 占用高，IO 少。
  适合：较长时间片、减少切换、SJF 等侧重吞吐量的策略。

- **IO 密集型**
  频繁读写磁盘/网络，CPU 空闲多，等待多。
  适合：高优先级、短时间片、尽快调度运行，提升响应速度。

## 4. 抢占式调度 vs 非抢占式
- **非抢占式**
  进程一旦开始运行就一直执行到结束或主动阻塞才释放 CPU。
  优点：切换少、开销小；缺点：响应慢，容易饥饿。

- **抢占式**
  操作系统可强行剥夺当前进程 CPU，分配给更高优先级/时间片到的进程。
  优点：实时性好、响应快；缺点：切换开销大，需处理同步问题。

现代操作系统（Linux、Windows）普遍使用**抢占式调度**。

## 1. 什么是阻塞IO、非阻塞IO、IO多路复用？
- **阻塞IO**：发起IO调用后，线程会一直阻塞等待，直到数据就绪并完成拷贝，期间线程不做任何事。
- **非阻塞IO**：线程发起IO后立即返回，若数据未就绪返回错误，线程需不断轮询询问，数据就绪后再完成拷贝。
- **IO多路复用**：一个线程可以同时监听多个文件描述符，当某个描述符就绪后再进行读写，实现单线程管理多路IO，select/poll/epoll 都属于这类。

## 2. select、poll、epoll 区别？
- **select**：采用数组存储文件描述符，有最大连接数限制，每次需遍历全部描述符，用户态与内核态数据拷贝开销大。
- **poll**：采用链表存储，无最大连接限制，其余缺点与select一致，仍需遍历全量描述符。
- **epoll**：使用事件驱动机制，只返回就绪描述符，无连接数限制，采用mmap减少用户态内核态拷贝，效率远高于前两者，是高并发场景首选。

## 3. 什么是零拷贝？
避免数据在用户态缓冲区与内核态缓冲区之间多次CPU拷贝，直接在内核空间完成数据流转，减少CPU开销与上下文切换。
常见实现：mmap、sendfile，常用于网络传输、文件读写场景提升性能。

## 4. 同步IO vs 异步IO区别？
- **同步IO**：线程需等待IO操作完成（数据就绪+数据拷贝）才能继续执行，阻塞IO、非阻塞IO、IO多路复用都属于同步IO。
- **异步IO**：线程发起IO请求后立即返回，内核完成数据就绪与拷贝后主动通知线程，全程线程不阻塞、不参与数据拷贝。

## 5. 文件系统基本概念：inode、软硬链接
- **inode**：存储文件元数据（权限、大小、时间、磁盘块指针等），不包含文件名，每个文件对应唯一inode号，通过inode定位文件数据。
- **硬链接**：与原文件指向同一个inode，删除原文件不影响硬链接，不能跨文件系统，不能指向目录。
- **软链接（符号链接）**：独立文件，存储原文件路径，类似快捷方式，删除原文件后失效，可跨文件系统、可指向目录。

## 6. 什么是文件描述符？
操作系统为进程打开的文件、socket、管道等分配的非负整数，用于标识操作对象，进程通过文件描述符对IO资源进行访问与管理。

## 7. 磁盘IO调度算法
- FCFS 先来先服务
- SSTF 最短寻道时间优先
- SCAN 扫描算法（电梯算法）
- C-SCAN 循环扫描
- LOOK、C-LOOK 优化版扫描

## 8. 什么是页缓存（Page Cache）？
操作系统将磁盘数据缓存到物理内存中，以页为单位管理，后续读写优先访问内存，减少磁盘IO，提升读写速度。
是文件读写性能优化的核心机制，修改数据会先写入页缓存，再由操作系统异步刷回磁盘。

## 1. 常用 Linux 命令用途
- **top**：实时查看系统整体负载、进程 CPU/内存占用、运行状态
- **ps**：查看进程快照，列出当前运行的进程信息
- **netstat**：查看网络连接、端口监听、路由表、网络协议统计
- **ss**：比 netstat 更高效的网络连接查看工具
- **lsof**：list open files，查看进程打开的文件、端口、目录等
- **df**：查看磁盘分区**总容量、已用、可用**（文件系统级别）
- **du**：查看文件/目录占用的磁盘空间大小
- **free**：查看系统内存、交换分区使用情况
- **dmesg**：查看内核环形缓冲区日志，用于排查硬件、驱动、内核级故障
- **kill**：向进程发送信号，用于终止、重启进程

## 2. 如何查看 CPU 占用、内存占用、句柄数、端口占用？
- **CPU 占用**：top、htop、ps aux、pidstat
- **内存占用**：free、top、ps aux、cat /proc/meminfo
- **句柄数**：lsof -p 进程号、ls /proc/进程号/fd、ulimit -a
- **端口占用**：netstat -tulnp、ss -tulnp、lsof -i:端口号

## 3. 什么是孤儿进程、僵尸进程？
- **孤儿进程**：父进程先退出，子进程还在运行，被 init/systemd 收养，无害。
- **僵尸进程**：子进程已退出，但父进程未调用 wait()/waitpid() 回收其 PCB 信息，进程表仍保留，会占用进程号，大量产生会导致无法创建新进程。

## 4. 什么是信号？常见信号有哪些？
信号是操作系统向进程发送的**软中断通知**，用于通知事件、终止进程等。
常见信号：
- **SIGINT(2)**：Ctrl + C 中断
- **SIGQUIT(3)**：Ctrl + \ 退出并产生 core
- **SIGKILL(9)**：强制杀死进程，不可捕获、不可忽略
- **SIGTERM(15)**：默认终止信号，可捕获处理
- **SIGSEGV(11)**：段错误，非法内存访问
- **SIGHUP(1)**：终端断开连接，常用于让进程重载配置
- **SIGPIPE(13)**：管道破裂，往已关闭的 socket 写数据时触发

## 5. 系统调用是什么？
应用程序请求操作系统内核提供服务的**接口**，如 open、read、write、fork、mmap 等。
是用户态程序进入内核态的主要方式，为上层屏蔽硬件差异，提供统一服务。

## 6. 用户态与内核态区别？如何切换？
- **用户态**：权限低，只能访问自身地址空间，不能直接操作硬件。
- **内核态**：权限高，可访问所有内存、硬件资源，执行特权指令。

切换场景：
- 系统调用（主动）
- 外部中断（硬件触发，如网卡、键盘）
- 异常（缺页中断、除零错误）

## 7. 什么是软中断、硬中断？
- **硬中断**：由硬件设备（网卡、磁盘）主动发出，打断 CPU 当前执行，快速处理硬件事件。
- **软中断**：内核模拟的中断机制，由内核触发，用于延后处理耗时逻辑（如网络收发包后半段），不阻塞硬件响应。

5. 什么是公平锁、非公平锁？

## 1. 什么是互斥锁、自旋锁、读写锁？
- **互斥锁（Mutex）**
  同一时刻只允许一个线程持有锁，其他线程会**阻塞挂起**，不消耗CPU。
- **自旋锁**
  线程拿不到锁时不会阻塞，而是**循环忙等**，不断尝试获取锁。
- **读写锁（共享排他锁）**
  允许多个线程同时读，但**写独占**；读多写少场景效率更高。

## 2. 信号量、管程、条件变量原理
- **信号量**
  用一个计数器表示可用资源数，`P`操作申请资源，`V`操作释放资源；可实现互斥与同步。
- **管程**
  把共享数据和对数据的操作封装在一起，保证同一时刻只有一个线程在内部执行，是高级同步机制。
- **条件变量**
  配合互斥锁使用，让线程在不满足条件时**等待阻塞**，条件满足后被唤醒，避免无效竞争。

## 3. 自旋锁和互斥锁适用场景？
- **自旋锁**：锁持有时间很短、多核CPU环境，避免线程切换开销。
- **互斥锁**：锁持有时间长、单核CPU，避免忙等浪费CPU。

## 4. 什么是CAS？和锁的关系？
- **CAS（Compare And Swap）**
  原子操作：先比较内存值是否等于预期值，相等则更新，否则重试。
- **和锁的关系**
  属于**无锁编程**思想，Java 中 `Atomic` 系列基于CAS实现；
  轻量、无阻塞，但高并发下会频繁自旋，且存在ABA问题。

## 5. 什么是公平锁、非公平锁？
- **公平锁**：线程按申请顺序排队获取锁，不会饥饿，但吞吐量低。
- **非公平锁**：线程可直接尝试抢占锁，不用排队，吞吐量更高，但可能产生饥饿。
Java 中 `synchronized` 是非公平锁，`ReentrantLock` 默认非公平，可指定为公平。

## 1. 什么是系统颠簸（thrashing）？
指内存不足时，进程频繁发生**缺页中断**，页面在内存和交换区之间不断换入换出，导致 CPU 大部分时间都在处理页交换，而不是执行进程，系统效率急剧下降。
根本原因：活跃内存总量 > 可用物理内存。

## 2. 什么是缓存一致性？
在多核 CPU 下，每个核心都有自己的高速缓存（L1/L2），当多个核心同时读写同一块内存数据时，会导致各缓存副本不一致。
缓存一致性协议（如 MESI）用于保证所有核心看到的内存数据是一致的。

## 3. 大端小端区别？
- **大端（Big Endian）**：数据的**高字节存放在低地址**，低字节存放在高地址。符合人类阅读习惯。
- **小端（Little Endian）**：数据的**低字节存放在低地址**，高字节存放在高地址。CPU 运算更方便。
x86 / ARM 常用小端；网络传输通常用大端。

## 4. 什么是管道、命名管道？
- **管道（匿名管道）**：半双工通信，只能用于**父子进程 / 兄弟进程**之间，没有实体文件，随进程创建销毁。
- **命名管道（FIFO）**：有实际磁盘文件节点，可用于**任意无关进程**间通信，遵循先进先出。

## 5. 什么是DMA？
直接内存访问（Direct Memory Access）。
允许硬件设备（网卡、磁盘）**不经过 CPU**，直接与内存进行数据传输，减少 CPU 中断和拷贝开销，大幅提升 IO 效率。

# 八、结合 Java 的 OS 问题（特色考点）
1. Java 线程与OS线程的关系（内核线程模型）
2. Java 锁与OS原语（mutex、futex）关系

5. epoll 与 Netty 关系

## 1. Java 线程与 OS 线程的关系（内核线程模型）
- 现代 JVM（HotSpot）使用 **一对一线程模型**：
  一个 Java 线程 直接映射 一个 **操作系统内核级线程（KLT）**。
- 线程调度、优先级、阻塞/唤醒完全由 OS 内核管理，JVM 不直接参与调度。
- 区别：
  - 轻量级线程/协程（如虚拟线程）是 M:N 模型，不一一对应 OS 线程。
  - 传统 Java 线程本质就是 OS 线程，创建成本高、切换开销大。

## 2. Java 锁与 OS 原语（mutex、futex）关系
- **synchronized**
  依赖 OS 的 **mutex（互斥锁）** 和 **futex** 实现。
  偏向锁 → 轻量级锁（自旋）→ 重量级锁（进入内核阻塞）。
- **ReentrantLock**
  基于 **AQS + CAS** 实现，竞争激烈时也会通过 **LockSupport.park** 进入内核阻塞。
- **futex**
  是 Linux 下的快速用户态互斥锁机制，先在用户态尝试，失败才进入内核态，减少系统调用开销。

## 3. JVM 堆、栈与 OS 虚拟内存关系
- **JVM 堆**
  是向 OS 申请的 **一大块连续虚拟内存**。
  通过 mmap/brk 分配，内部由 JVM 自己管理（Eden/Survivor/Old）。
  GC 只是 JVM 层面的内存回收，不一定立即归还给 OS。
- **JVM 栈**
  每个线程对应一个 **OS 线程栈**，是 OS 分配的虚拟内存区域。
  栈大小由 `-Xss` 指定，超出会抛 StackOverflowError。
- 整体：
  JVM 内存 = OS 虚拟内存的一部分，受进程地址空间与物理内存限制。

## 4. 为什么 Java 会出现句柄过多、线程过多导致 OOM/卡死？
- **线程过多**
  每个线程都要分配栈空间（几MB），线程太多会耗尽 **虚拟内存/物理内存**，触发 OOM。
  同时大量线程导致频繁上下文切换，CPU 飙升，系统卡死。
- **句柄过多（文件描述符 FD）**
  Socket、文件、管道都会占用 FD，超过系统限制（ulimit）会抛 Too many open files。
  大量 FD 会占用内存、影响 IO 效率，甚至导致服务不可用。

## 5. epoll 与 Netty 关系
- Netty 是 **高并发 NIO 框架**，底层依赖 OS 的 IO 多路复用。
- Linux 下 Netty 默认使用 **epoll** 作为 IO 多路复用器（NIOEventLoop）。
- 作用：
  单线程监听大量连接，只处理就绪事件，避免阻塞和轮询浪费 CPU。
- 对比：
  Windows 用 select，Linux 优先 epoll，这也是 Netty 在 Linux 下高并发性能极强的原因。
