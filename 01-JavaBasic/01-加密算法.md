## **密码加密技术体系**树状图

以下是**密码加密技术体系**的树状图（层级结构），从顶层目标到底层具体算法，清晰展示现代密码学中用于**密码存储与验证**的核心方法：

```
密码加密（Password Encryption / Hashing）
│
├── 1. **不可逆哈希（Hashing）** ← 推荐！用于密码存储
│   │
│   ├── 1.1 **通用加密哈希函数**（不推荐用于密码！）
│   │   ├── MD5
│   │   └── SHA-1
│   │
│   └── 1.2 **专用密码哈希函数**（推荐！慢速、加盐、抗暴力破解）
│       │
│       ├── 1.2.1 **PBKDF2**（Password-Based Key Derivation Function 2）
│       │   └── 基于 HMAC（如 HMAC-SHA256）
│       │
│       ├── 1.2.2 **bcrypt**
│       │   └── 基于 Blowfish 加密算法
│       │
│       ├── 1.2.3 **scrypt**
│       │   └── 内存-hard 算法（抗 ASIC/GPU 攻击）
│       │
│       └── 1.2.4 **Argon2** ← 当前最推荐（2015 年 Password Hashing Competition 获胜者）
│           ├── Argon2d（侧重侧信道攻击防护）
│           ├── Argon2i（侧重 GPU 攻击防护）
│           └── Argon2id（混合模式，推荐默认使用）
│
├── 2. **可逆加密（Encryption）** ← ❌ 不应用于原始密码存储！
│   │
│   ├── 2.1 对称加密（Symmetric）
│   │   ├── AES
│   │   ├── DES / 3DES（已不安全）
│   │   └── ChaCha20
│   │
│   └── 2.2 非对称加密（Asymmetric）
│       ├── RSA
│       └── ECC（Elliptic Curve Cryptography）
│
└── 3. **辅助安全机制**
    │
    ├── 3.1 **加盐（Salt）** ← 必须！防止彩虹表攻击
    │   └── 随机唯一盐值（每个用户独立）
    │
    ├── 3.2 **胡椒（Pepper）** ← 可选增强
    │   └── 全局密钥（存储在代码或 KMS 中，非数据库）
    │
    └── 3.3 **自适应工作因子（Work Factor）**
        ├── bcrypt: cost 参数（如 12）
        ├── PBKDF2: 迭代次数（如 600,000）
        ├── scrypt: CPU/内存成本参数
        └── Argon2: time, memory, parallelism 参数
```

### 🔑 关键说明：

#### ✅ **为什么只推荐“不可逆哈希”？**

- 密码只需**验证**（比对哈希值），无需**还原**。
- 即使数据库泄露，攻击者也无法直接获取明文密码。

#### ⚠️ **为什么 MD5/SHA-1 不安全？**

- 速度太快 → 易被暴力破解/彩虹表攻击。
- 已存在大量碰撞漏洞。

#### 🛡️ **专用密码哈希函数的优势：**

| 特性         | 作用                               |
| ------------ | ---------------------------------- |
| **慢速计算** | 增加暴力破解时间（如 0.1~1 秒/次） |
| **内置加盐** | 自动处理盐值，避免开发者遗漏       |
| **可调强度** | 通过参数适应硬件升级               |

#### 💡 **最佳实践（2025年）：**

1. **首选 Argon2id**（若环境支持）

2. **次选 bcrypt**（广泛支持，如 Spring Security 默认）

3. 避免自己实现加密逻辑

    → 使用成熟库：

   - Java: `spring-security-crypto`, `jargon2`
   - Python: `passlib`
   - Node.js: `bcrypt`, `argon2`

> 📌 **重要原则**：
> **永远不要存储用户原始密码！**
> **永远不要使用可逆加密存储密码！**

### 1. **什么是加密算法？在 Java 中加密算法的分类有哪些？**

**定义：**

- 加密算法是一种将原始信息（明文）通过数学方法转换为不可读形式（密文）的技术，以保护信息安全。
- 加密过程通常需要密钥，解密则需要密钥或相关算法。

**Java 中加密算法分类：**

1. **对称加密（Symmetric Encryption）**
   - 加密和解密使用相同密钥。
   - 常用算法：AES、DES、3DES、Blowfish。
2. **非对称加密（Asymmetric Encryption）**
   - 加密和解密使用不同密钥（公钥/私钥）。
   - 常用算法：RSA、DSA、ECC。
3. **摘要算法（Hash）**
   - 单向哈希函数，输入数据生成固定长度的摘要，不可逆。
   - 常用算法：MD5、SHA-1、SHA-256。
4. **数字签名（Digital Signature）**
   - 用于验证消息完整性和发送者身份。
   - 通常结合非对称加密和摘要算法实现。



#### 1. 什么是加密算法？

加密算法是一种通过数学变换将原始数据（明文）转换为不可读形式（密文）的技术，目的是保护数据的机密性、完整性或真实性。只有通过特定的密钥或方法才能将密文还原为明文（解密），从而防止未授权访问或篡改。

#### 2. Java 中加密算法的分类

Java 中加密算法主要通过 `java.security` 和 `javax.crypto` 包提供支持，按功能和原理可分为以下几类：

##### （1）对称加密算法（Symmetric Cryptography）

- **特点**：加密和解密使用 **同一把密钥**，密钥需要在通信双方之间安全传递；加密速度快，适合处理大量数据。  
- **常用算法**：  
  - **AES（Advanced Encryption Standard）**：目前最常用的对称加密算法，支持 128/192/256 位密钥，安全性高，替代了 DES。  
  - **DES（Data Encryption Standard）**：早期标准，56 位密钥，安全性较低，已逐渐被淘汰。  
  - **3DES（Triple DES）**：对 DES 进行三次加密，增强安全性，但效率低于 AES。  
  - **RC4**：流加密算法，速度快，但安全性存在漏洞，不推荐用于敏感场景。  
- **Java 实现**：通过 `Cipher` 类配合 `SecretKey` 实现，例如 `Cipher.getInstance("AES/ECB/PKCS5Padding")`。  

##### （2）非对称加密算法（Asymmetric Cryptography）

- **特点**：使用 **一对密钥（公钥和私钥）**，公钥可公开，私钥需保密；用公钥加密的数据只能用对应私钥解密，反之亦然；加密速度较慢，适合小数据（如密钥交换）。  
- **常用算法**：  
  - **RSA**：应用最广泛的非对称算法，支持密钥长度 1024/2048/4096 位，可用于加密和数字签名。  
  - **DSA（Digital Signature Algorithm）**：仅用于数字签名，不支持加密。  
  - **ECC（Elliptic Curve Cryptography）**：基于椭圆曲线数学，相同安全级别下密钥长度更短（如 256 位 ECC 等效于 3072 位 RSA），效率更高。  
- **Java 实现**：通过 `KeyPairGenerator` 生成密钥对，`Cipher` 类加密解密（如 `Cipher.getInstance("RSA")`）。  

##### （3）哈希算法（Hash Algorithm）

- **特点**：将任意长度数据转换为固定长度的哈希值（摘要），**不可逆**（无法从哈希值还原明文）；相同输入必产生相同哈希值，不同输入大概率产生不同哈希值（抗碰撞性）。主要用于数据完整性校验和密码存储（加盐哈希）。  
- **常用算法**：  
  - **MD5**：生成 128 位哈希值，安全性较低（存在碰撞漏洞），不推荐用于敏感场景。  
  - **SHA 系列**：  
    - SHA-1：160 位哈希值，安全性不足，已被淘汰；  
    - SHA-256/SHA-512：属于 SHA-2 系列，分别生成 256/512 位哈希值，安全性高，广泛使用；  
    - SHA-3：新一代哈希标准，与 SHA-2 算法原理不同，兼容性稍差。  
  - **HMAC（Hash-based Message Authentication Code）**：结合哈希算法和密钥，用于验证数据完整性和真实性（如 HMAC-SHA256）。  
- **Java 实现**：通过 `MessageDigest` 类（如 `MessageDigest.getInstance("SHA-256")`）或 `Mac` 类（HMAC）实现。  

##### （4）数字签名算法

- **特点**：基于非对称加密和哈希算法，用于验证数据的真实性和完整性（防止伪造和篡改）。发送方用私钥签名，接收方用公钥验签。  
- **常用算法**：  
  - **RSA 签名**：结合 RSA 和哈希算法（如 SHA256withRSA）。  
  - **DSA 签名**：基于 DSA 算法，配合 SHA 哈希。  
  - **ECDSA（Elliptic Curve Digital Signature Algorithm）**：基于 ECC 的签名算法，效率高于 RSA 签名。  
- **Java 实现**：通过 `Signature` 类（如 `Signature.getInstance("SHA256withRSA")`）实现。  

#### 总结

Java 中的加密算法按功能可分为：  
- **对称加密**（AES 为主）：适合大量数据加密，速度快；  
- **非对称加密**（RSA、ECC 为主）：适合密钥交换和签名，安全性高；  
- **哈希算法**（SHA-256 为主）：适合完整性校验和密码存储；  
- **数字签名**（基于非对称和哈希）：适合身份验证和防篡改。  

实际开发中常结合使用（如 HTTPS：对称加密传输数据 + 非对称加密交换对称密钥 + 数字签名验证证书）。

### 4. **Java 中 `javax.crypto` 和 `java.security` 包分别用于什么？**

1. **`javax.crypto`**

   - 核心类：

     - `Signature`：数字签名
     - `SecureRandom`：安全随机数生成器

**总结：**

- `javax.crypto` 偏重实际加密解密操作。
- `java.security` 偏重密钥管理、签名和摘要。





在 Java 中，`javax.crypto` 和 `java.security` 是处理加密、解密、签名、认证等安全相关操作的核心包，两者分工不同但协同工作，共同支撑 Java 的安全框架。

#### **1. `java.security` 包：核心安全基础与通用安全服务**

`java.security` 是 Java 安全体系的基础包，主要提供 **通用安全框架、核心安全服务和基础安全机制**，涵盖密钥管理、消息摘要、数字签名、访问控制等底层功能。  

##### 核心功能与类：

1. **消息摘要（哈希算法）**  
   - 提供 `MessageDigest` 类，支持 MD5、SHA-1、SHA-256 等哈希算法，用于生成数据的不可逆摘要（如校验数据完整性）。  
   ```java
   MessageDigest digest = MessageDigest.getInstance("SHA-256");
   byte[] hash = digest.digest("data".getBytes()); // 生成 SHA-256 哈希值
   ```

2. **数字签名与验证**  
   - 提供 `Signature` 类，支持 RSA、DSA、ECDSA 等签名算法，用于数据的真实性和完整性验证（私钥签名，公钥验签）。  
   ```java
   Signature signature = Signature.getInstance("SHA256withRSA");
   signature.initSign(privateKey); // 私钥初始化签名
   signature.update(data);
   byte[] sign = signature.sign(); // 生成签名
   ```

3. **密钥对生成与管理**  
   - 提供 `KeyPairGenerator` 类，用于生成非对称加密的密钥对（公钥 + 私钥），支持 RSA、DSA、ECC 等算法。  
   ```java
   KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
   keyGen.initialize(2048); // 密钥长度 2048 位
   KeyPair keyPair = keyGen.generateKeyPair(); // 生成 RSA 密钥对
   ```

4. **安全随机数**  
   - 提供 `SecureRandom` 类，用于生成加密级别的随机数（比 `java.util.Random` 更安全，适合生成密钥、盐值等）。  
   ```java
   SecureRandom random = new SecureRandom();
   byte[] salt = new byte[16];
   random.nextBytes(salt); // 生成 16 字节随机盐值
   ```

5. **访问控制与安全策略**  
   
   - 提供 `Permission`、`Policy` 等类，用于定义和管理代码的访问权限（如限制文件读写、网络访问等），支撑 Java 的安全沙箱机制。  

#### **2. `javax.crypto` 包：加密解密专用功能**

`javax.crypto` 是 `java.security` 的扩展包，专注于 **加密、解密、密钥交换等具体密码学操作**，主要提供对称加密、非对称加密、密码块模式、流加密等实现。  

##### 核心功能与类：

1. **对称加密与解密**  
   - 提供 `Cipher` 类，是加密解密的核心工具，支持 AES、DES、3DES 等对称算法，以及 RSA 等非对称算法的加密解密操作。  
   ```java
   Cipher cipher = Cipher.getInstance("AES/ECB/PKCS5Padding");
   cipher.init(Cipher.ENCRYPT_MODE, secretKey); // 对称密钥初始化加密模式
   byte[] cipherText = cipher.doFinal(plainText); // 加密明文
   ```

2. **密钥生成与密钥材料**  
   - 提供 `KeyGenerator` 类，用于生成对称加密的密钥（如 AES 密钥）。  
   ```java
   KeyGenerator keyGen = KeyGenerator.getInstance("AES");
   keyGen.init(256); // AES 密钥长度 256 位
   SecretKey secretKey = keyGen.generateKey(); // 生成 AES 密钥
   ```
   - 提供 `SecretKey`、`SecretKeySpec` 等类，用于表示和封装对称密钥。  

3. **密码块链与填充模式**  
   
- 支持多种加密模式（如 ECB、CBC、GCM）和填充方式（如 PKCS5Padding），通过 `Cipher.getInstance()` 的参数指定（如 `AES/CBC/PKCS5Padding`）。  
  
4. **密钥交换**  
   
- 提供 `KeyAgreement` 类，支持 Diffie-Hellman 等密钥交换算法，用于双方在不直接传递密钥的情况下协商出共享密钥。  
  
5. **消息认证码（MAC）**  
   - 提供 `Mac` 类，支持 HMAC 等算法（如 HMAC-SHA256），结合密钥和哈希函数验证数据的完整性和真实性。  
   ```java
   Mac mac = Mac.getInstance("HmacSHA256");
   mac.init(secretKey); // 用密钥初始化
   byte[] macBytes = mac.doFinal(data); // 生成消息认证码
   ```

#### **两者的关系与区别**

| 维度         | `java.security` 包                          | `javax.crypto` 包                          |
|--------------|---------------------------------------------|---------------------------------------------|
| **定位**     | 基础安全框架，提供通用安全服务（如哈希、签名、密钥对生成）。 | 专注于加密解密操作，提供对称/非对称加密、密钥交换等具体实现。 |
| **依赖关系** | 是 `javax.crypto` 的基础，`javax.crypto` 中的类常依赖 `java.security` 的核心类（如 `Key`、`Provider`）。 | 扩展 `java.security` 的功能，专注于密码学操作的实现。 |
| **核心场景** | 消息摘要、数字签名、密钥对生成、安全随机数、访问控制。 | 对称加密、非对称加密、密钥生成（对称）、MAC 认证、密钥交换。 |

#### **总结**

- `java.security` 是 Java 安全的“基础平台”，提供通用安全服务（哈希、签名、密钥对、随机数等），是所有安全操作的基石。  
- `javax.crypto` 是“加密解密专用工具”，基于 `java.security` 扩展，专注于对称/非对称加密、密钥交换等具体密码学操作。  

实际开发中，两者通常结合使用（例如：用 `java.security` 生成 RSA 密钥对，用 `javax.crypto` 的 `Cipher` 类进行 RSA 加密解密）。

### 5. **什么是数字签名？它与加密有什么关系？**

**定义：**

- 数字签名是对信息的 **完整性、真实性、不可否认性** 的保障。
- 发送方用 **私钥** 对信息摘要加密生成签名，接收方用 **公钥** 验证签名。

**实现步骤：**

1. 生成消息摘要（如 SHA-256）。
2. 用私钥加密摘要 → 得到数字签名。
3. 接收方用公钥解密签名，得到摘要，再与原文计算摘要对比 → 验证完整性和身份。

**关系与区别：**

- 加密保证数据保密性（数据内容不可读）。
- 数字签名保证数据完整性与身份认证（可读，但可验证来源）。
- 数字签名通常结合非对称加密和摘要算法使用。

**Java 示例：**

```java
// 签名
Signature signature = Signature.getInstance("SHA256withRSA");
signature.initSign(privateKey);
signature.update(message.getBytes());
byte[] signBytes = signature.sign();

// 验签
signature.initVerify(publicKey);
signature.update(message.getBytes());
boolean valid = signature.verify(signBytes);
```

### 1. **Java 中常用的对称加密算法有哪些？**

**对称加密算法**：加密和解密使用相同密钥，速度快，适合大数据量加密。

**常见算法及特点：**

| 算法     | 密钥长度       | 模式                | 特点                             |
| -------- | -------------- | ------------------- | -------------------------------- |
| DES      | 56 位          | ECB/CBC             | 老算法，不安全，已逐渐弃用       |
| 3DES     | 112/168 位     | ECB/CBC             | DES 的增强版，但仍被认为不够安全 |
| AES      | 128/192/256 位 | ECB/CBC/CFB/OFB/GCM | 高安全性，高性能，广泛使用       |
| Blowfish | 32–448 位      | ECB/CBC             | 可变密钥长度，适合嵌入式         |
| RC4      | 40–2048 位     | 流加密              | 速度快，但已被认为不安全         |

**面试要点：**

- AES 是目前最常用、最安全的对称加密算法。
- DES 已不推荐使用，3DES 仅在兼容场景使用。
- 对称加密优点：速度快、适合大数据加密；缺点：密钥管理困难。

------

### 2. **如何使用 AES 算法在 Java 中加密和解密字符串？**

**示例代码：**

```java
import javax.crypto.Cipher;
import javax.crypto.KeyGenerator;
import javax.crypto.SecretKey;
import javax.crypto.spec.SecretKeySpec;
import java.util.Base64;

public class AESExample {
    public static void main(String[] args) throws Exception {
        String content = "Hello, AES!";
        
        // 生成密钥
        KeyGenerator keyGen = KeyGenerator.getInstance("AES");
        keyGen.init(128); // 密钥长度
        SecretKey secretKey = keyGen.generateKey();
        byte[] keyBytes = secretKey.getEncoded();
        SecretKeySpec keySpec = new SecretKeySpec(keyBytes, "AES");

        // 加密
        Cipher cipher = Cipher.getInstance("AES/ECB/PKCS5Padding");
        cipher.init(Cipher.ENCRYPT_MODE, keySpec);
        byte[] encrypted = cipher.doFinal(content.getBytes());
        String encryptedBase64 = Base64.getEncoder().encodeToString(encrypted);
        System.out.println("加密结果: " + encryptedBase64);

        // 解密
        cipher.init(Cipher.DECRYPT_MODE, keySpec);
        byte[] decrypted = cipher.doFinal(Base64.getDecoder().decode(encryptedBase64));
        System.out.println("解密结果: " + new String(decrypted));
    }
}
```

**面试要点：**

- AES 使用 `Cipher` 类进行加解密。
- 模式（Mode）和填充（Padding）必须匹配，否则解密失败。
- ECB 模式容易泄露数据模式，生产环境推荐使用 CBC 或 GCM。

------

### 3. **什么是密钥长度？AES 支持哪些常用密钥长度？**

**密钥长度（Key Length）：**

- 指加密算法中密钥的比特数。密钥越长，安全性越高，但加密解密速度可能略慢。

**AES 支持的常用密钥长度：**

- 128 位：最快，安全性高，广泛使用
- 192 位：中等安全性
- 256 位：最高安全性，但需要安装 Java JCE 无限权限策略（JDK 8 以前）

**面试要点：**

- AES 密钥长度与安全性成正比，长度不足可能被暴力破解。
- AES-128 已被认为足够安全，适合大多数应用。

------

### 4. **对称加密在实际应用中可能存在什么安全风险？**

1. **密钥管理困难**
   - 加密和解密使用相同密钥，若密钥泄露，数据安全性将丧失。
   - 解决方法：结合非对称加密安全传输密钥。
2. **模式问题**
   - ECB 模式加密容易泄露数据模式（相同明文块加密结果相同）。
   - 推荐使用 CBC/GCM，并配合随机 IV。
3. **填充攻击**
   - 不正确的填充或解密方式可能导致 Padding Oracle 攻击。
4. **重放攻击**
   - 相同明文使用相同密钥和 IV 加密，会产生相同密文，容易被攻击者分析。
5. **随机性不足**
   - IV 或密钥生成不随机，会降低安全性。

**面试要点：**

- 对称加密适合数据加密，但需要安全管理密钥、使用安全模式、加入 IV、配合消息认证码（MAC）确保完整性。

### 1. **Java 中常用的非对称加密算法有哪些？**

**非对称加密（Asymmetric Encryption）**：加密和解密使用不同密钥（公钥/私钥），通常用于安全通信和数字签名。

**常见算法及特点：**

| 算法                | 密钥类型          | 特点                       | 应用场景                       |
| ------------------- | ----------------- | -------------------------- | ------------------------------ |
| RSA                 | 公钥/私钥         | 最常用，基于大素数分解     | 密钥交换、加密小数据、数字签名 |
| DSA                 | 私钥签名/公钥验证 | 数字签名算法               | 数字签名                       |
| ECC（椭圆曲线加密） | 公钥/私钥         | 密钥短、速度快、强度高     | 移动端加密、数字签名           |
| ElGamal             | 公钥/私钥         | 安全性高，基于离散对数问题 | 研究和加密通信                 |

**面试要点：**

- 非对称加密适合密钥交换、数字签名等场景，不适合大数据量加密。
- RSA 最常用，ECC 在移动端和新系统中越来越流行。

------

### 2. **RSA 算法的加密原理是什么？如何在 Java 中实现？**

**原理：**

1. 生成一对密钥（公钥和私钥）。
2. 公钥加密 → 私钥解密。
3. 也可以私钥加密 → 公钥解密（数字签名用途）。
4. 基于大素数分解困难性保证安全。

**Java 实现示例：**

```java
import java.security.*;
import javax.crypto.Cipher;
import java.util.Base64;

public class RSAExample {
    public static void main(String[] args) throws Exception {
        String content = "Hello, RSA!";

        // 生成密钥对
        KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
        keyGen.initialize(2048); // 推荐2048位
        KeyPair keyPair = keyGen.generateKeyPair();
        PublicKey publicKey = keyPair.getPublic();
        PrivateKey privateKey = keyPair.getPrivate();

        // 加密
        Cipher cipher = Cipher.getInstance("RSA/ECB/PKCS1Padding");
        cipher.init(Cipher.ENCRYPT_MODE, publicKey);
        byte[] encrypted = cipher.doFinal(content.getBytes());
        String encryptedBase64 = Base64.getEncoder().encodeToString(encrypted);
        System.out.println("加密结果: " + encryptedBase64);

        // 解密
        cipher.init(Cipher.DECRYPT_MODE, privateKey);
        byte[] decrypted = cipher.doFinal(Base64.getDecoder().decode(encryptedBase64));
        System.out.println("解密结果: " + new String(decrypted));
    }
}
```

**面试要点：**

- 公钥加密、私钥解密 → 数据保密
- 私钥加密、公钥解密 → 数字签名
- RSA 对大数据量加密效率低，一般用于加密对称密钥。

------

### 3. **非对称加密为什么通常比对称加密慢？**

**原因：**

1. **数学复杂度高**
   - 非对称加密基于大素数分解或椭圆曲线，计算量大。
   - 对比对称加密，AES/DES 的矩阵运算简单。
2. **密钥长度长**
   - RSA 推荐 2048 位，ECC 推荐 256 位；对比 AES 128 位。
   - 密钥长度越长，加解密时间越慢。
3. **通常用于小数据量加密**
   - 实际应用中非对称加密用于加密对称密钥，而大数据量使用对称加密。

**面试要点：**

- 非对称加密慢，常与对称加密结合使用（混合加密方案）。

------

### 4. **在实际应用中如何使用非对称加密进行密钥交换？**

**应用场景：**

- 安全通信，如 HTTPS、TLS。
- 客户端和服务器共享对称密钥，用非对称加密传输。

**实现流程（RSA + AES 举例）：**

1. **服务器生成 RSA 密钥对**（公钥/私钥）。
2. **客户端获取服务器公钥**。
3. **客户端生成随机 AES 对称密钥**。
4. **客户端用服务器公钥加密 AES 密钥** → 发送给服务器。
5. **服务器用私钥解密** → 获取 AES 密钥。
6. **双方使用 AES 密钥加密大数据通信**。

**面试要点：**

- 结合非对称加密（安全传输密钥） + 对称加密（高效数据加密）。
- 这种方案称为 **混合加密（Hybrid Encryption）**，是现代安全通信标准（如 TLS）的核心。

### 1. **常用的摘要算法有哪些？MD5、SHA-1、SHA-256 有何区别？**

**摘要算法（Hash）特点：**

- 单向不可逆，固定长度输出。
- 用于验证数据完整性、生成数字签名。

**常用算法及对比：**

| 算法    | 输出长度 | 安全性 | 特点         | 应用                         |
| ------- | -------- | ------ | ------------ | ---------------------------- |
| MD5     | 128 位   | 已破坏 | 快速但易碰撞 | 文件完整性校验（非安全场景） |
| SHA-1   | 160 位   | 较弱   | 碰撞攻击风险 | 旧版数字签名/验证            |

### 2. **Java 中如何计算字符串或文件的 Hash 值？**

**示例：计算字符串 Hash（SHA-256）**

```java
import java.security.MessageDigest;
import java.util.Base64;

public class HashExample {
    public static void main(String[] args) throws Exception {
        String input = "Hello, Hash!";
        MessageDigest md = MessageDigest.getInstance("SHA-256");
        byte[] digest = md.digest(input.getBytes());
        String hashBase64 = Base64.getEncoder().encodeToString(digest);
        System.out.println("SHA-256 Hash: " + hashBase64);
    }
}
```

**示例：计算文件 Hash（SHA-256）**

```java
import java.io.FileInputStream;
import java.security.MessageDigest;

public class FileHashExample {
    public static void main(String[] args) throws Exception {
        MessageDigest md = MessageDigest.getInstance("SHA-256");
        try (FileInputStream fis = new FileInputStream("test.txt")) {
            byte[] buffer = new byte[1024];
            int n;
            while ((n = fis.read(buffer)) != -1) {
                md.update(buffer, 0, n);
            }
        }
        byte[] digest = md.digest();
        System.out.println(javax.xml.bind.DatatypeConverter.printHexBinary(digest));
    }
}
```

**面试要点：**

- `MessageDigest` 用于计算 Hash，`update` 可分块处理文件。
- 输出通常可用 Base64 或 Hex 表示。

### 3. **数字签名如何实现消息完整性和不可否认性？**

**目的：**

- **完整性**：消息未被篡改
- **不可否认性**：发送者不能否认发送过消息

**实现原理：**

1. 发送方：
   - 对消息计算摘要（Hash）
   - 用私钥加密摘要 → 数字签名
2. 接收方：
   - 使用公钥解密数字签名 → 得到摘要
   - 对原消息计算摘要，比较是否一致 → 验证完整性
   - 公钥验证成功 → 证明签名者身份

**面试要点：**

- 数字签名结合 **Hash + 非对称加密**。
- 保证消息未被篡改，同时验证发送者身份。

------

### 4. **Java 中如何使用 `Signature` 类实现数字签名？**

**示例代码：**

```java
import java.security.*;
import java.util.Base64;

public class DigitalSignatureExample {
    public static void main(String[] args) throws Exception {
        String message = "Hello, Digital Signature!";

        // 1. 生成密钥对
        KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
        keyGen.initialize(2048);
        KeyPair keyPair = keyGen.generateKeyPair();
        PrivateKey privateKey = keyPair.getPrivate();
        PublicKey publicKey = keyPair.getPublic();

        // 2. 签名
        Signature signature = Signature.getInstance("SHA256withRSA");
        signature.initSign(privateKey);
        signature.update(message.getBytes());
        byte[] signBytes = signature.sign();
        String signBase64 = Base64.getEncoder().encodeToString(signBytes);
        System.out.println("数字签名: " + signBase64);

        // 3. 验签
        signature.initVerify(publicKey);
        signature.update(message.getBytes());
        boolean isValid = signature.verify(signBytes);
        System.out.println("验签结果: " + isValid);
    }
}
```

**面试要点：**

- `Signature` 类封装了数字签名流程（摘要 + 非对称加密）。
- 签名算法选择通常是 `"SHA256withRSA"` 或 `"SHA256withECDSA"`。
- 验签时必须使用与签名时相同的算法和公钥。

##  五、应用与安全实践

------

### 1. **在 Java 中如何安全存储密码？为什么不要直接用 MD5/MD5+盐？**

**安全存储密码方法：**

1. **使用强哈希函数 + 盐（Salt）**
   - 推荐算法：`PBKDF2`、`BCrypt`、`SCrypt`
   - 特点：
     - 多轮迭代增加破解难度
     - 每个用户生成独立盐，防止彩虹表攻击
2. **Java 示例（BCrypt）**

```java
import org.mindrot.jbcrypt.BCrypt;

public class PasswordExample {
    public static void main(String[] args) {
        String password = "mySecret123";

        // 加密
        String hashed = BCrypt.hashpw(password, BCrypt.gensalt(12));
        System.out.println("加密后: " + hashed);

        // 验证
        boolean matched = BCrypt.checkpw(password, hashed);
        System.out.println("验证结果: " + matched);
    }
}
```

**为什么不要直接用 MD5 或 MD5+盐？**

- MD5 已经被证明容易碰撞，单线程暴力破解非常快。
- MD5+固定盐仍可能被彩虹表攻击。
- 现代安全实践推荐使用专门的密码哈希算法（如 BCrypt、PBKDF2）。

------

### 2. **什么是加密算法模式（Mode）和填充（Padding）？常见模式有哪些？**

**模式（Mode）**

- 对称加密分块加密算法需要 **模式** 来处理明文分块。
- 常见模式：
  - **ECB（电子密码本）**：每个块独立加密，安全性低
  - **CBC（密码分组链接）**：每个块与前一块相关，需 IV，安全性高
  - **CFB/OFB（流模式）**：块加密变流加密，适合流式数据
  - **GCM（Galois/Counter Mode）**：支持认证加密（AEAD），安全且支持消息完整性

**填充（Padding）**

- 数据长度可能不是块大小倍数，需要填充：
  - **PKCS5Padding / PKCS7Padding**：常用填充方式
  - **NoPadding**：不填充，要求数据长度正好是块大小倍数

**Java 示例（AES CBC 模式 + PKCS5 填充）**

```java
Cipher cipher = Cipher.getInstance("AES/CBC/PKCS5Padding");
```

**面试要点：**

- 模式决定加密安全性和数据结构泄露情况。
- 填充方式必须匹配，否则解密失败。

------

### 3. **实际项目中如何结合对称加密、非对称加密和数字签名实现安全通信？**

**场景：客户端 ↔ 服务器安全通信**

**步骤（混合加密 + 签名）：**

1. **非对称加密用于密钥交换**
   - 服务器生成 RSA 公私钥对
   - 客户端生成随机 AES 密钥
   - 客户端用服务器公钥加密 AES 密钥 → 发送给服务器
   - 服务器用私钥解密 → 双方共享 AES 密钥
2. **对称加密用于数据传输**
   - 使用共享 AES 密钥加密大数据（性能高）
   - 使用安全模式（CBC/GCM）和 IV
3. **数字签名用于验证数据完整性和身份**
   - 发送方对消息生成摘要
   - 用私钥签名摘要 → 发送消息 + 签名
   - 接收方用公钥验证签名 → 确认完整性和身份

**Java 混合方案示意图：**

```
客户端                    服务器
  |                        |
生成 AES 密钥                 |
  |                        |
AES 密钥用公钥加密 ----------> 解密获得 AES 密钥
  |                        |
AES 加密消息 ---------------> AES 解密消息
签名消息 ------------------> 验签
```

**面试要点：**

- **混合加密**：非对称加密保护密钥，对称加密传输数据。
- **签名**：保证数据未篡改，验证身份。
- **安全通信标准**：TLS/HTTPS 就是这种混合方案的实际应用。
