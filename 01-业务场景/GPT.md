# GPT

在 **Java 大厂面试**中，业务场景题通常不是考 API，而是考 **架构能力 + 业务抽象能力 + 高并发设计能力**。
如果系统化整理，可以分成 **12 大类业务场景题类型**。

下面只 **列举题目分类（不回答答案）**。

------

## 一、缓存类业务场景

### 常见题型

1. 如何设计 **热点数据缓存**？
2. 如何解决 **缓存穿透**？
3. 如何解决 **缓存击穿**？
4. 如何解决 **缓存雪崩**？
5. 如何保证 **缓存与数据库一致性**？
6. 如何设计 **多级缓存（本地缓存 + Redis）**？
7. Redis 宕机如何保证系统可用？
8. 如何设计 **缓存预热机制**？
9. 如何实现 **缓存自动失效策略**？

------

## 二、秒杀 / 高并发抢购系统

### 常见题型

1. 如何设计 **秒杀系统架构**？
2. 如何防止 **超卖**？
3. 秒杀系统如何做 **流量削峰**？
4. 如何防止 **重复下单**？
5. 如何做 **秒杀库存扣减**？
6. 如何解决 **秒杀瞬时百万请求**？
7. 如何设计 **秒杀限流机制**？

------

## 三、分布式锁场景

### 常见题型

1. 如何实现 **分布式锁**？
2. Redis 分布式锁如何防止 **死锁**？
3. 如何解决 **锁过期问题**？
4. 如何保证 **锁的高可用**？
5. 如何实现 **可重入分布式锁**？
6. Redisson 锁的原理是什么？

------

## 四、分布式事务场景

### 常见题型

1. 如何设计 **分布式事务系统**？
2. 订单创建 + 库存扣减如何保证一致？
3. 如何实现 **最终一致性**？
4. **Seata AT 模式原理**
5. 如何设计 **事务补偿机制**？
6. 如何避免 **事务悬挂问题**？

## 五、消息队列业务场景

常涉及
消息可靠性、顺序性、重复消费等。

### 常见题型

1. 如何设计 **订单异步处理系统**？
2. 如何保证 **消息不丢失**？
3. 如何保证 **消息不重复消费**？
4. 如何保证 **消息顺序消费**？
5. MQ 消费失败如何处理？
6. 如何设计 **消息重试机制**？

常见 MQ：

- Apache RocketMQ
- Apache Kafka
- RabbitMQ
