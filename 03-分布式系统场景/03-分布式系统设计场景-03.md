# 继续补全｜25个冷门+压轴系统设计场景3

全部是**容易被问倒、进阶八股、中大厂追问**题型，短小好背。

## 25. 接口防重设计

场景：重复提交、网络重试、表单重复点击
方案：

1. 令牌机制：下单前申请Token，提交校验一次性销毁
2. 请求维度：`requestId+用户ID` 组合Key加Redis防重
3. 数据库：唯一索引、业务状态机
4. 兜底：短时间分布式锁

## 26. 推送系统设计（APP/站内信）

难点：亿级用户、在线长连接、离线消息
架构：

1. 长连接网关：Netty/WebSocket 维持在线
2. 在线：实时推送
3. 离线：MQ积压+入库，上线拉取
4. 分级推送：全员推送、标签推送、精准用户

## 27. 分布式链路追踪设计

核心流程：

1. 入口生成 `TraceId`，全局透传
2. 微服务传递 `SpanId`、调用层级
3. 采集日志、耗时、异常
4. 存储：Elasticsearch，展示：SkyWalking/Pinpoint
   作用：排查分布式调用超时、接口报错、性能瓶颈

## 28. 热点数据/大V流量削峰

场景：网红商品、爆款文章、大V动态
问题：热点Key打爆缓存、DB压力暴增
方案：

1. 本地缓存+Caffeine 二级缓存
2. 热点Key 永不过期
3. 流量隔离：大V单独队列、资源隔离
4. 静态化、CDN、页面缓存

## 29. 文件秒传/断点续传

核心思路：

1. 文件MD5唯一指纹
2. 服务端先查MD5，存在直接秒传
3. 分片上传：按固定分片切割
4. 断点：记录已上传分片编号，续传跳过已完成

## 30. 定时流水号生成（每日重置）

场景：每日订单号、流水号 `202604150001`
方案：

1. Redis 每日key自增，自动过期
2. 数据库号段每日预分配
3. 防止并发重复：Lua原子自增

## 31. 库存扣减设计（精准）

三种场景全覆盖：

1. 普通库存：Redis+Lua 预扣 + 最终落库
2. 秒杀库存：限时隔离、超卖防护
3. 退回库存：延时释放、防止二次超卖
   核心：**预扣+异步落库+定时对账**

## 32. 敏感词过滤系统

方案：

1. 离线词库 + **Trie字典树** 快速匹配
2. 轻量：Redis缓存敏感词
3. 高阶：AI语义识别
4. 替换脱敏、拦截拒绝发布

## 33. 黑白名单系统

1. 黑名单：IP/用户ID/设备号 Redis 存储
2. 白名单：内部测试、灰度用户
3. 网关层统一拦截，不进入业务服务
4. 动态刷新、实时生效

## 34. 多租户SaaS设计

三种方案：

1. 独立库：隔离最高、成本高
2. 共享库、独立表：中等隔离
3. **共享库共享表**：租户ID 字段隔离（最常用）
   公共字段：`tenant_id` 数据隔离

## 35. 异地多活设计

难点：数据同步、异地容灾、故障切换
方案：

1. 单元化部署，就近接入
2. 异地数据同步、双向复制
3. 故障自动切换、流量调度
4. 最终一致性优先，保障可用性

## 36. 数据对账系统

场景：支付、财务、订单、流水
设计：

1. 日切任务：每日凌晨批量对账
2. 三方对账：本地流水 vs 第三方回调
3. 差异数据入库、人工+自动补偿
4. 幂等重试、防重复对账

## 37. 短时延实时监控系统

指标：QPS、错误率、响应时间、JVM指标
流程：
服务埋点 → 指标采集 → 时序数据库(InfluxDB/Prometheus) → 可视化告警
告警：短信、钉钉、邮件实时通知

## 38. 防止订单超卖/少卖

核心组合拳：

1. 库存原子扣减：Lua
2. 数据库悲观/乐观锁兜底
3. 下单幂等、一人一单限制
4. 库存定时校对，修复缓存偏差

## 39. 版本灰度发布设计

方案：

1. 网关层路由规则
2. 灰度规则：用户ID、IP、百分比、白名单
3. 流量染色、环境隔离
4. 平滑上线，异常一键回滚

## 40. 海量定时任务调度

场景：千万级延时任务、批量到期处理
方案：

1. 时间轮算法（TimeWheel）
2. 分片调度+分布式任务框架
3. MQ延时队列承载超大批量
4. 分片隔离，避免单点压力

## 41. 冷热数据分离

场景：订单、日志、历史数据
设计：

1. 热数据：MySQL+Redis 高频访问
2. 冷数据：归档到廉价存储、分库分表归档
3. 定时迁移、查询自动路由冷热库

## 42. 服务优雅下线

步骤：

1. 注册中心摘除实例，停止接收新流量
2. 等待存量请求处理完毕
3. 停止定时任务、消息消费
4. 关闭线程池、资源释放、平滑退出

## 43. 大促全链路压测

方案：

1. 压测流量标记隔离
2. 影子库、影子表，不污染真实数据
3. 全链路瓶颈分析：CPU/内存/GC/慢SQL
4. 提前扩容、降级预案、限流阈值调整

## 44. 数据脱敏设计

场景：手机号、身份证、地址展示

1. 前端脱敏展示
2. 后端存储加密（AES）
3. 数据库脱敏、视图屏蔽
4. 日志脱敏拦截，防止敏感信息泄露

## 45. 集群时间同步问题

分布式痛点：机器时间不一致
解决：

1. 统一NTP时间服务器同步
2. 业务尽量使用**服务端时间**
3. 雪花算法配置时间校验，防时钟回拨

## 终极汇总（45道系统设计全覆盖）

1. 高并发类：秒杀、热点、限流、削峰
2. 数据一致类：分布式事务、库存、对账、幂等
3. 通信解耦类：RPC、网关、MQ、推送
4. 存储缓存类：多级缓存、冷热分离、分库分表
5. 调度延时类：延时队列、定时任务、过期关闭
6. 安全风控类：防重、防刷、敏感词、黑白名单
7. 运维高可用：灰度、多活、优雅下线、链路追踪
