# coze & dify

### 一、基础概念题（必背）

#### Coze（字节扣子）
1. **Coze 是什么？**  
字节跳动推出的**零代码/低代码 AI 智能体搭建平台**，支持可视化配置、插件扩展、对话流程编排，可快速生成 Agent 并一键发布到抖音、飞书等生态。

2. **Coze 核心能力有哪些？**  
- 对话编排：可视化设计多轮对话流程；  
- 插件系统：API/数据库/自定义函数调用；  
- 知识库：文档上传+检索增强生成（RAG）；  
- 多模态：支持文本/图片/语音输入输出；  
- 生态集成：一键发布到抖音、飞书、豆包等。

3. **Coze 和 Dify 的核心区别？**  
- Coze：**闭源 SaaS、零代码、C 端生态强**，适合快速做简单 Agent；  
- Dify：**开源可私有化、低代码、企业级能力强**，适合复杂工作流与私有部署。

#### Dify
1. **Dify 是什么？**  
开源的**LLM 应用开发平台**，集成模型接入、Prompt 编排、RAG、Workflow、Agent、工具插件、API 发布与观测能力，支持从原型到生产的全链路。

2. **Dify 核心应用场景？**  
企业知识库问答、智能客服、报告生成、流程自动化、内部工具助手、数据查询助手等，尤其擅长 **RAG+工作流+工具调用** 结合的复杂场景。

3. **Dify 与 LangChain 的区别？**  
LangChain：代码级框架，灵活但需开发，适合深度定制；  
Dify：平台化工具，**可视化编排+开箱即用+运营观测**，适合快速落地企业级应用。

### 二、平台能力与原理题（高频）
#### Coze
1. **Coze 对话状态管理和传统规则引擎（如有限状态机）有何不同？**  
传统：硬编码状态节点，切换僵硬，难处理模糊输入；  
Coze：**大模型驱动上下文理解**，动态更新状态，自动识别用户意图与信息完整性，支持自然语言交互。

2. **如何用 Coze 设计一个旅行攻略 Agent？**  
- 人设：设定为旅行规划师；  
- 对话流程：收集目的地→天数→预算→偏好（美食/景点）；  
- 插件：调用天气 API、景点数据库、酒店预订接口；  
- 知识库：上传目的地攻略文档做 RAG；  
- 发布：一键到抖音/飞书，支持多轮追问。

3. **Coze 如何防止 LLM 生成有害内容？**  
- 前置过滤：输入敏感词拦截；  
- 提示词约束：人设中明确“拒绝违规请求”；  
- 内容审核：输出对接字节安全审核接口；  
- 权限控制：插件调用白名单，限制高危操作。

#### Dify
1. **Dify 的 RAG 实现原理？如何优化召回率？**  
原理：文档解析→切片→向量化→存储（向量库）→检索→排序→生成引用。  
优化：  
- 混合检索：**向量+关键词（ES）** 多路召回；  
- 切片策略：按标题/段落分割，避免语义断裂；  
- 查询扩展：同义词/相关词补充，解决稀疏查询；  
- 重排序：用 Cross-BERT 精排，提升相关性。

2. **Dify 的 Workflow 与 ChatFlow 区别？**  
- ChatFlow：对话触发，线性流程，适合简单问答；  
- Workflow：**可视化复杂编排**，支持分支/循环/并行/子流程，可定时/API触发，适合企业级自动化。

3. **Dify 多模型兼容如何实现？私有化部署如何保障数据安全？**  
- 多模型：统一抽象接口，封装不同模型参数（temperature、top_p），适配同步/异步调用；  
- 数据安全：**本地部署（数据不出企业）+HTTPS传输+存储加密+细粒度权限控制**。

### 三、实操与场景题（面试重点）
#### Coze
1. **场景：用 Coze 做电商售后 Agent，需处理退货/退款/查物流，如何设计？**  
- 开场白：引导用户选择业务类型；  
- 分支流程：  
  - 查物流：调用快递 API，输入单号返回轨迹；  
  - 退货：收集订单号→原因→上传凭证→对接售后系统；  
  - 退款：校验订单状态→金额→调用支付接口；  
- 兜底：无法处理时转人工，记录会话日志。

2. **Coze 插件开发步骤？如何调用本地 HTTP 接口？**  
- 步骤：新建插件→定义入参/出参→配置请求地址/方法/头信息→测试→发布；  
- 本地调用：Coze 云端无法直接访问内网，需**内网穿透（如 ngrok）/部署 Coze 私有化/配置企业代理**。

#### Dify
1. **场景：用 Dify 搭建企业知识库问答系统，要求支持 PDF/Word 上传、精准检索、引用来源，如何实现？**  
- 知识库配置：创建知识库→上传文档→自动切片+向量化（默认用 text-embedding）；  
- 应用类型：选“知识库问答”→绑定知识库；  
- Prompt 设计：约束“仅用知识库内容回答，标注来源页码”；  
- 检索设置：混合检索（向量+关键词）+重排序，提升精准度；  
- 发布：生成 API 或 Web 页面，嵌入企业系统。

2. **Dify 工作流无法访问本地接口的原因？如何解决？**  
原因：Dify 云端部署，默认无法访问内网/本地 127.0.0.1；  
解决：  
- 内网穿透：ngrok/frp 暴露本地端口；  
- 私有化部署：Dify 部署在企业内网；  
- 配置代理：Dify 服务器添加企业内网路由；  
- 本地服务绑定 0.0.0.0，允许外部访问。

### 四、对比与高阶题（加分项）
1. **Coze 与 Dify 各自适合的团队/场景？**  
- Coze：**非技术人员、C 端产品、快速验证、字节生态**；  
- Dify：**技术团队、企业级应用、复杂流程、私有部署、数据安全敏感**。

2. **Dify 生产环境如何控制 Token 成本？**  
- 缓存：重复查询结果缓存（如 Redis）；  
- 限流：按用户/应用限制请求数；  
- 模型选择：短文本用小模型（如 Qwen-7B），长文本用大模型；  
- Prompt 优化：精简提示词，减少冗余 Token；  
- 监控：跟踪 Token 消耗，异常告警。

3. **Coze/Dify 多智能体设计要点？**  
- 角色分工：明确 Agent 职责（如咨询/售后/技术）；  
- 记忆共享：通过会话变量/全局状态传递上下文；  
- 通信机制：工具调用/消息队列实现 Agent 间协作；  
- 冲突处理：优先级规则+人工兜底，避免回答矛盾。
