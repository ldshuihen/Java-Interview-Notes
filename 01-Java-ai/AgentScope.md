# AgentScope

 最新详解（2026-05，含1.0→1.2核心升级）

AgentScope 是阿里通义实验室推出的**企业级开源多智能体框架**，主打**透明可控、原生多智能体、生产级工程化**，覆盖从原型到分布式部署的全链路。2025-09 发布 1.0，2026 持续迭代至 **1.2 稳定版**，新增语音、Java 生态、Serverless 运行时等关键能力。

---

## 一、核心定位与设计哲学

### 1）解决的行业痛点

- **协作混乱**：多 Agent 通信无标准、数据不同步、重复劳动/任务断层
- **开发低效**：重复造记忆/工具/日志轮子，难聚焦业务
- **工程化缺失**：不可观测、难复现、难扩展，上线后故障难定位

### 2）设计哲学（四大原则）

- **透明可控**：全程可观测、可中断、可人工干预，拒绝黑盒
- **原生多智能体**：消息驱动（msghub）、显式 A2A 协议、灵活编排
- **模型无关**：一次编写，适配通义千问/GPT/DeepSeek 等
- **模块化乐高**：组件解耦、可插拔、易定制

## 二、整体架构（三层：Core + Runtime + Studio）

### 1）AgentScope Core（核心框架）

“智能体操作系统”，模块化四层结构：
- **消息层（msghub）**：统一消息路由、多模态（文/图/音）、异步/同步、A2A 协议
- **模型层**：统一模型接口、异步调用、多模态、推理模型支持
- **记忆层**：短期对话窗口、长期记忆（向量库/数据库）、记忆压缩、RAG 原生集成
- **工具层**：统一工具协议（MCP/Function Call）、工具沙箱、安全隔离

**核心范式：ReAct（推理→行动）**，支持：
- 工具调用、思维链、反思、规划
- 人机协同（human-in-the-loop）、实时中断/干预

定位：**智能体操作系统**，整体划分为**四层模块化架构**，全域遵循 **ReAct 推理行动范式**

- 能力：统一消息路由分发，兼容文本/图片/语音多模态消息

- 价值：实现多Agent解耦协作，支持跨进程、跨机器分布式通信

##### 3. 记忆层

- 短期记忆：自动维护对话上下文、智能截断超长上下文

## 四、核心能力详解

### 1）消息驱动的多智能体协作（msghub）

- **统一消息结构**：`{id, sender, receiver, content, type, timestamp, metadata}`
- **通信模式**：一对一、广播、组播、流水线、星型拓扑
- **A2A 协议**：标准化 Agent 发现、注册、通信，支持跨进程/跨机器
- **案例**：“总控 Agent → 调研 Agent → 写作 Agent → 审核 Agent” 流水线

#### 一、核心定位

**msghub = 多Agent统一消息总线**
是 AgentScope 实现**去中心化/中心化多智能体协作**的核心通信中枢，所有Agent交互**不走函数调用，只走消息流转**。

#### 二、标准消息结构体（原生字段）

```json
{
  "id": "消息唯一ID",
  "sender": "发送方Agent名称",
  "receiver": "接收方Agent名称/广播标识",
  "content": "消息主体内容",
  "type": "消息类型(任务/结果/通知/指令)",
  "timestamp": "时间戳",
  "metadata": "附加元数据(任务ID、优先级、上下文ID)"
}
```

#### 三、五大通信模式

1. **一对一单播**：指定sender→指定receiver，精准点对点传话
2. **全局广播**：一条消息所有接入Hub的Agent全部接收
3. **分组组播**：只发给指定成员小组，隔离业务域
4. **流水线串行**：任务按固定顺序流转（调度→执行→汇总）
5. **星型拓扑**：中心主控Agent下发任务，子Agent统一回传结果

#### 四、A2A 智能体互联协议

1. **自动发现**：Agent启动自动注册进msghub，无需手动绑定
2. **跨域互通**：支持**同进程、多进程、跨机器、分布式集群**通信
3. **统一规约**：消息格式、调用格式、回调格式全局统一
4. **解耦最强**：修改任意Agent逻辑，不改动其他协作代码

#### 五、经典流水线实战流程

**总控Agent → 调研Agent → 写作Agent → 审核Agent**
1. 总控发**任务消息**推入消息总线
2. 调研Agent监听消息，领取任务执行，回传调研结果消息
3. 写作Agent接收调研消息，生成文稿消息
4. 审核Agent监听文稿消息，完成校验返回最终结果
5. 全程**无硬编码调用**，只靠消息订阅分发

#### 六、Python极简运行代码

```python
from agentscope import ReActAgent, init, msghub

init(model_configs={"qwen": {"type": "dashscope", "model_name": "qwen-turbo"}})

# 创建四类智能体
controller = ReActAgent(name="总控Agent", model_config_name="qwen")
researcher = ReActAgent(name="调研Agent", model_config_name="qwen")
writer = ReActAgent(name="写作Agent", model_config_name="qwen")
checker = ReActAgent(name="审核Agent", model_config_name="qwen")

# 接入同一消息总线，自动互通
with msghub(participants=[controller, researcher, writer, checker]):
    controller.run("完成一篇行业分析报告")
```

1. **彻底解耦**：Agent之间零依赖

3. **灵活编排**：随时增减Agent、调整流转顺序
4. **易分布式部署**：单机直接改成集群多节点协作
5. **天然支持人机介入**：人工可向总线推送干预消息

#### 一、一句话理解

**AgentScope 的记忆 = 短期记忆（记住刚说过的话）+ 长期记忆（记住所有历史/知识库）**
让 Agent 像人一样：
- 记得**最近对话**
- 记得**很久以前的事**
- 记得**外部知识**

#### 二、短期记忆（Short-term Memory）

**作用：保存最近几轮对话上下文，让 Agent 知道“现在在聊什么”**

##### 核心特点

- 自动维护**对话窗口**
- 自动**截断超长文本**，防止模型超限
- 每轮对话自动追加
- 开箱即用，**不需要你写代码**

##### 工作方式

```
用户：你好
Agent：你好！
用户：我想写报告
Agent：好的，什么主题？
```
这些内容都会被**短期记忆**自动保存。

---

#### 三、长期记忆（Long-term Memory）

**作用：保存历史对话、知识库、文件、网页等永久/半永久信息**
让 Agent 拥有“超能力记忆”。

##### 支持存储

- 历史对话
- 文档（PDF/Word/TXT）
- 网页内容
- 结构化知识
- 自定义数据

##### 底层支持

- **向量库**：Milvus、FAISS、Chroma
- **数据库**：MySQL、PostgreSQL
- **向量检索 + 关键词检索** 双引擎

---

#### 四、记忆管理（AgentScope 最强特性）

让记忆**智能、不混乱、不溢出**：

- **重要性评分**：自动判断哪些记忆更关键
- **遗忘机制**：自动清理不重要记忆
- **记忆压缩**：把长文本提炼成核心要点
- **RAG 原生集成**：检索 → 增强 → 回答 一键完成

### 3）工具调用与安全沙箱

- **统一工具接口**：兼容 OpenAI Function Call、MCP 协议
- **内置工具**：代码解释器、文件操作、网络请求、数据库查询、浏览器控制
- **安全沙箱**：工具执行与 Agent 逻辑隔离，支持 Docker/VM 级隔离
- **自定义工具**：装饰器 `@tool` 快速注册，自动生成参数校验与文档

---

#### 一、一句话定位

**工具调用 = 给 Agent 装上“手脚”，让它能执行真实任务；
安全沙箱 = 给 Agent 戴上“手铐脚镣”，让它只能在安全范围内干活。**

---

#### 二、统一工具接口

AgentScope 做了**一层标准化封装**，不管底层模型是啥，工具调用写法完全统一。

##### 支持两种行业标准

1. **OpenAI Function Call**
   最通用的工具调用格式，所有大模型都兼容。
2. **MCP 协议（Model Control Protocol）**
   阿里推出的**企业级工具调用协议**，更安全、更可控、支持权限/审计/限流。

##### 好处

- 一套代码，通吃所有大模型
- 工具定义一次，到处能用
- 自动参数校验、自动生成文档

#### 四、安全沙箱（企业级核心）

**这是 AgentScope 区别于其他框架的最大亮点之一：安全可控。**

##### 沙箱作用

- 工具执行**完全与主程序隔离**
- Agent 不能删文件、不能改系统、不能访问内网
- 所有操作可审计、可拦截、可记录

##### 支持的隔离级别

1. **进程隔离**（轻量）
2. **Docker 容器隔离**（标准）
3. **VM 虚拟机隔离**（最高安全）

##### 企业必备能力

- 权限控制
- 操作审计日志
- 危险指令拦截
- 文件访问白名单
- 网络访问白名单

#### 五、自定义工具（超级简单）

只用一个 **`@tool` 装饰器**，就能把任何函数变成 Agent 可调用的工具。

##### 代码示例（可直接复制运行）

```python
from agentscope import tool
from agentscope.agents import ReActAgent

# 1. 自定义工具：只需要加 @tool
@tool
def add(a: int, b: int) -> int:
    """加法计算
    Args:
        a: 数字1
        b: 数字2
    Returns:
        a + b
    """
    return a + b

@tool
def get_weather(city: str) -> str:
    """获取城市天气
    Args:
        city: 城市名称
    Returns:
        天气情况
    """
    return f"{city} 今天晴天，25℃"

# 2. 创建 Agent 并绑定工具
agent = ReActAgent(
    name="助手",
    model_config_name="qwen",
    tools=[add, get_weather]  # 直接传入工具列表
)

# 3. 运行，Agent 会自动判断何时调用工具
agent.run("北京天气如何？100+200等于多少？")
```

##### 自动实现的能力（你完全不用写）

- 工具描述生成
- 参数类型校验
- 异常捕获
- 执行日志记录
- 沙箱安全包装

## 五、与 LangChain/LangGraph 对比（核心差异）

| 维度     | AgentScope                    | LangChain/LangGraph            |
| -------- | ----------------------------- | ------------------------------ |
| 设计定位 | 企业级多智能体生产框架        | 单 Agent 为主，多 Agent 弱支持 |
| 通信机制 | 原生 msghub+A2A，标准化       | 无统一通信，依赖自定义队列     |
| 可观测性 | 全链路可视化、Studio 控制台   | 基础日志，无原生可视化         |
| 部署形态 | 原生支持 Serverless/K8s/Java  | 依赖第三方部署，Java 支持弱    |
| 安全隔离 | 内置工具沙箱、权限控制        | 无原生沙箱，需额外集成         |
| 适用场景 | 复杂多 Agent 协作、企业级落地 | 快速原型、单 Agent 应用        |

一句话总结：**LangChain 是灵活的“特种部队”，AgentScope 是纪律严明的“集团军”**。
