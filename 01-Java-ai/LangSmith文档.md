# LangSmith

LangSmith 是 LangChain 推出的**LLM应用全生命周期可观测平台**，定位为“AI 应用的 APM+调试器+评估系统”，核心解决大模型应用“黑盒不可控”问题，让 Chain/Agent/RAG 可追踪、可调试、可评估、可优化。

---



## 一、核心定位与价值

- **框架无关**：支持 LangChain、LangGraph、原生 OpenAI/Anthropic 调用，也可通过 OpenTelemetry 接入非 Python 服务。
- **全链路追踪**：记录一次请求从输入到输出的**完整调用链（Trace）**，含模型调用、工具调用、Prompt 渲染、中间结果、耗时、Token 消耗、成本。
- **生产级监控**：实时仪表盘（延迟、错误率、成本、Token）+ 告警 + 智能聚类分析。
- **评估闭环**：数据集管理、自动评估（LLM-as-judge/自定义指标）、人工反馈、版本对比。
- **协作与迭代**：Prompt 版本管理、Playground 可视化调试、团队协作评审。



---

## 二、核心概念（必懂）

- **Trace（调用链）**：一次用户请求的**完整执行树**，由多个 Run 组成。
- **Run（执行单元）**：Trace 的最小节点，如 LLM 调用、工具调用、Chain 调用、检索（Retriever）调用，记录输入/输出/参数/耗时/状态。
- **Project（项目）**：隔离不同应用/环境的 Trace，便于管理与权限控制。
- **Dataset（数据集）**：用于评估的样本集合，可由生产 Trace 导出或手动上传。
- **Experiment（实验）**：在数据集上运行不同 Prompt/模型/配置，对比效果。

### 1. Trace 调用链

**一次完整用户请求的全流程日志**
用户发一个问题，从进入程序 → 走提示词 → 调用大模型 → 调用工具 → 拼接结果 → 最终回复，**整条完整流程**就叫一条 Trace。
可视化呈现**树形结构**，能看清每一步怎么走的。

### 2. Run 执行单元

Trace 里面**最小独立执行节点**，Trace 由无数 Run 拼接而成。
常见 Run 类型：

- LLM Run：大模型调用
- Chain Run：链路执行
- Tool Run：工具调用
- Retriever Run：知识库检索
  每个 Run 单独记录：入参、出参、耗时、Token、报错信息。

### 3. Project 项目

用来**分类隔离**线上、测试、本地不同业务AI应用。
例：智能客服项目、RAG知识库项目、办公Agent项目分开建项目，日志互不混杂，方便团队管理、权限划分、环境区分。

### 4. Dataset 评估数据集

专门用来**测评AI效果**的测试题库。
来源两种：

1. 线上真实用户对话 Trace 直接导出
2. 手动批量编写标准问答样本
   作用：统一标准，批量测试模型好坏。

### 5. Experiment 实验

在同一个 Dataset 数据集上，**对照测试不同方案**。
比如：

- 测试Prompt A vs Prompt B
- 测试GPT-3.5 vs 国产模型
- 测试不同检索策略
  自动生成对比报表，直观看出哪个版本效果更好，快速迭代优化。

---

### 极简关系总结

**一次用户请求 = 1条 Trace**
**一条 Trace = 多个 Run 节点**
**所有 Trace 按 Project 项目归类**
**拿 Dataset 题库，跑 Experiment 实验做效果评测**

## 三、七大核心功能

### 1. 可观测性（Tracing+监控+Insights）

- **自动追踪**：一行代码开启，自动捕获 LLM/Chain/Agent/Tool 的全链路。
- **细节可视化**：每步输入/输出、Prompt 原文、工具参数、错误栈、延迟、Token、成本。
- **监控大盘**：P50/P99 延迟、错误率、成本趋势、Top 慢调用、Top 错误。
- **智能分析**：Trace 聚类、失败模式识别、根因定位、优化建议。
- **断点调试**：支持在任意 Run 前后打断点，单步执行，适合复杂 Agent 调试。

#### 1. 自动追踪

无需手动写日志，仅配置环境变量即可全局开启
自动抓取 **大模型、链式调用、智能代理、自定义工具** 所有执行流程，全程无侵入接入。

#### 2. 细节可视化

全流程信息一目了然，调试核心依据：

- 每一步**输入内容、最终输出结果**
- 真实渲染后的**完整Prompt**
- 工具调用传参、返回数据
- 程序报错堆栈、调用耗时
- 消耗输入/输出Token、接口调用成本

#### 3. 监控大盘

企业线上运维核心能力

- 统计请求**中位延迟P50、高耗时延迟P99**
- 全局错误请求占比、接口失败率
- 每日/每月模型调用成本走势
- 筛选最慢请求、高频报错请求，快速定位问题

#### 4. 智能分析

平台自动聚合数据分析

- 把相似异常调用归类聚合
- 总结高频失败原因、归纳问题模式
- 自动给出性能优化、Prompt优化、成本优化建议

#### 5. 断点调试

复杂Agent专属调试功能
可在任意**最小执行单元Run**前后暂停流程，单步运行排查
精准定位：工具调用异常、思考逻辑错误、多轮循环卡死、RAG检索失效等疑难问题。

---

#### 核心作用总结

把**黑盒大模型应用**彻底透明化，开发调试排错、线上运维监控、性能成本优化一站式搞定。

### 2. 评估系统（Evaluation）

- **离线评估**：在 Dataset 上批量运行，自动打分（相关性、正确性、有害性等）。
- **在线评估**：生产环境实时对 Trace 打分，监控质量变化。
- **LLM-as-Judge**：用 GPT-4 等作为评委，自动评估输出质量。
- **人工反馈**：支持标注团队直接在平台打分、评论、修正输出。
- **版本对比**：同一 Dataset 对比不同 Prompt/模型/代码版本的效果。

#### 1. 离线评估

提前准备好**测试数据集Dataset**，批量跑完整问答流程，自动批量打分评判。
评判维度：内容相关性、答案正确性、语句合规性、有无有害内容等，适合开发阶段批量验收效果。

#### 2. 在线评估

线上生产环境实时监测用户真实对话，对每条请求链路实时评分，持续监控AI回答质量波动，及时发现线上效果下滑问题。

#### 3. LLM-as-Judge 大模型裁判

借助更强的大模型（如GPT-4）充当人工评委，自动对AI输出内容进行打分、判错、挑漏洞，无需人工逐条审核，实现全自动质量校验。

#### 4. 人工反馈

平台支持业务标注人员直接在追踪记录里手动评分、写评语、修正错误回答，收集真实业务标准答案，用来优化模型与提示词。

#### 5. 版本对比

使用同一套测试数据集，横向对比**不同提示词、不同大模型、不同业务代码版本**的输出效果，直观选出最优方案，快速迭代优化。

**核心作用**：建立AI应用量化评价标准，告别主观感受，用数据衡量回答好坏，稳定上线质量。

### 3. Prompt 工程与协作

- **Playground**：可视化编辑、调试 Prompt，实时预览输出，支持多模型对比。
- **版本管理**：Prompt 历史版本回溯、分支管理、变更记录。
- **团队协作**：Prompt Canvas 支持多人在线编辑、评论、建议，直接合并优化。

#### 1. Playground 在线调试工作台

可视化编写提示词，即时运行查看模型返回结果；可一键切换多款大模型，**同问题横向对比不同模型回答效果**，快速择优选型。

#### 2. Prompt 版本管理

自动留存提示词每一次修改记录，支持历史版本回滚、分支区分、查看修改明细，避免优质提示词丢失，方便迭代回退。

#### 3. 团队协作开发

依托Prompt画布实现多人协同编辑，成员可在线批注、提修改意见、评审文案，统一团队Prompt规范，高效合力优化提示词，告别本地文件传来传去。

**核心作用**：规范化管理提示词，降低调试成本，实现团队协同共建优质提示词体系。

### 4. Agent 部署（Agent Server）

- **一键部署**：将 Agent/Chain 打包为可扩展的 HTTP 服务，支持负载均衡、自动扩缩容。
- **托管/自托管**：支持 LangSmith 云托管、私有化部署或混合架构，满足合规（HIPAA/SOC2/GDPR）。

#### 1. 一键部署

直接把写好的**智能体Agent、业务链路Chain**快速打包成标准HTTP接口服务
自带负载均衡、弹性扩缩容能力，并发高时自动扩容，空闲自动缩容，适配线上业务流量。

#### 2. 双部署模式

- **云托管**：LangSmith云端直接托管运行，不用自己维护服务器，开箱即用
- **私有化部署**：部署在企业内网/自有服务器，数据不外流
  满足行业数据安全、隐私合规、审计等各类合规要求。

**核心作用**：快速把本地调试完成的AI应用，低成本上线投产，正式对外提供服务。

### 5. Studio（可视化 IDE）

- **无代码/低代码**：拖拽式设计 Agent 流程，连接 LLM、工具、检索、条件分支。
- **端到端测试**：在可视化界面直接运行、调试、迭代整个应用。

#### 1. 无代码/低代码搭建

可视化拖拽编排智能体流程，直接拖拽**大模型、工具调用、知识库检索、逻辑分支**等组件，连线组合就能搭建完整Agent业务流程，大幅减少手写代码量，新手也能快速搭建AI应用。

#### 2. 端到端全流程测试

在可视化界面内直接发起请求，一站式完成运行、排错、流程调整，全程可视化查看每一步执行走向，快速迭代优化整体业务逻辑，不用反复本地调试。

**核心作用**：降低AI应用开发门槛，可视化梳理复杂业务流程，快速原型搭建与流程验证。

## 四、快速上手（5 步）

### 1. 安装与配置

```bash
pip install -U langsmith langchain-openai
```

```python
import os
# 注册：https://smith.langchain.com/
os.environ["LANGSMITH_API_KEY"] = "ls_xxx"
os.environ["LANGSMITH_TRACING"] = "true"  # 开启追踪
os.environ["LANGSMITH_PROJECT"] = "my-agent-app"  # 项目名
os.environ["OPENAI_API_KEY"] = "sk-xxx"
```

### 2. 追踪 LLM 调用

```python
from langchain_openai import ChatOpenAI
from langsmith import traceable

llm = ChatOpenAI(model="gpt-3.5-turbo")

@traceable  # 自动追踪此函数
def ask(question: str):
    return llm.invoke(question).content

ask("什么是 LangSmith？")
# 访问 https://smith.langchain.com/ 查看 Trace
```

### 3. 追踪 Agent 全链路

```python
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate

@tool
def search(keyword: str) -> str:
    """联网搜索"""
    return f"{keyword} 的搜索结果"

tools = [search]
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是助手，可使用工具"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}")
])
agent = create_openai_tools_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools)

agent_executor.invoke({"input": "LangSmith 有什么功能？"})
# LangSmith 会记录：思考→工具调用→结果→回答 全链路
```

## 五、典型使用场景

1. **调试复杂 Agent**：定位工具调用失败、参数错误、循环调用、输出不符合预期的根因。
2. **RAG 系统优化**：追踪检索→重排→生成全链路，定位召回不准、幻觉、引用错误。
3. **Prompt 迭代**：A/B 测试不同 Prompt，对比效果，快速收敛最优版本。
4. **生产监控告警**：实时监控延迟、成本、错误率、输出质量，异常自动告警。
5. **合规审计**：全链路日志留存，支持审计、溯源、合规检查。

## 六、LangSmith vs 传统日志

- **传统日志**：文本化、无结构、难追踪、难关联、无法可视化、无法自动评估。
- **LangSmith**：结构化 Trace 树、全链路可视化、自动元数据（Token/成本/耗时）、一键评估、智能分析。
