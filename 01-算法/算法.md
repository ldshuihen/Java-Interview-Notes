# 算法面试「链表」

链表是面试**最常考、最容易写崩**的数据结构，核心就考**指针操作、边界处理、快慢指针、递归**。下面按**高频优先级**整理，直接背这套足够应对绝大多数面试。

## 一、最核心通用技巧（所有题都离不开）
1. **虚拟头结点（dummy node）**
   - 解决：头结点可能被删除/修改的情况
   - 作用：统一逻辑，不用特判头结点
2. **快慢指针（双指针）**
   - 找中点、找环、找倒数第k个、重排链表
3. **递归**
   - 反转链表、合并有序链表、复杂删除
4. **pre / cur / next 三指针**
   - 绝大多数迭代写法标配

---

## 二、高频题型 & 对应解法（必刷）
### 1. 找环 / 判断环入口
- 方法：**快慢指针（Floyd判圈）**
- 快指针走2步，慢走1步：
  - 相遇 = 有环
  - 相遇后，一个从头、一个从相遇点同速走 → 相遇即**环入口**

### 2. 找链表中点
- 方法：**快慢指针**
- 快2慢1，快到末尾时，慢就在中点
- 用途：回文链表、链表归并排序、重排链表

### 3. 倒数第K个节点
- 方法：**快慢指针（前后指针）**
- 快先走k步，然后快慢同走
- 快到尾，慢就是**倒数第k**

### 4. 反转链表（必考之王）
- 两种写法都要会：
  1. **迭代（三指针 pre/cur/next）**
  2. **递归（从后往前反转）**
- 变种：**反转区间[left,right]**、**k个一组反转**

### 5. 合并两个有序链表
- 迭代 + dummy
- 递归（极简代码）
- 变种：合并k个有序链表（堆/分治）

### 6. 删除节点
- 删除重复元素（有序/无序）
- 删除给定值节点
- 删除**倒数第n个**（前后指针）
- 删除**给定节点（不给头）**：复制下一节点值，再删下一节点

### 7. 回文链表
- 方法：**找中点 + 反转后半段 + 双指针比较**
- 空间O(1)最优解

### 8. 重排链表
- L0→L1→…→Ln-1→Ln 重排为 L0→Ln→L1→Ln-1…
- 步骤：找中点 → 反转后半 → 合并交错

### 9. 两数相加（链表版）
- 从低位/高位相加，处理进位
- 高位版本要先**反转链表**

### 10. 链表排序
- 最优：**归并排序**
- 步骤：找中点 → 递归拆分 → 有序合并

---

## 三、面试常问的“连环炮”问题
1. 如何判断链表有环？怎么找入口？
2. 如何O(1)空间找中点？

4. 删除头结点为什么要用dummy？
5. 回文链表能不能不用栈/数组？
6. k个一组反转边界怎么处理？

# 算法面试：**二分查找 + 二分排序** 

二分是面试**最容易细节写错、边界卡死**的高频点，**90% 二分题都逃不出下面这套模板+题型**，直接背稳过。

## 一、二分查找（核心：边界写法 + 变种）
### 1. 三大必背模板（直接默写）
#### （1）标准二分：找**存在性**
```python
left = 0, right = len(nums)-1
while left <= right:
    mid = (left + right) // 2
    if nums[mid] == target: return mid
    elif nums[mid] < target: left = mid + 1
    else: right = mid - 1
return -1
```
**特点**：`left <= right`、`left=mid+1`、`right=mid-1`

#### （2）找**第一个 ≥ target**（左边界/下界）
```python
left = 0, right = n
while left < right:
    mid = (left + right) // 2
    if nums[mid] >= target: right = mid
    else: left = mid + 1
return left
```
**用途**：插入位置、重复数第一个位置、统计小于target数量

#### （3）找**最后一个 ≤ target**（上边界）
```python
left = 0, right = n
while left < right:
    mid = (left + right + 1) // 2  # 向上取整防死循环
    if nums[mid] <= target: left = mid
    else: right = mid - 1
return left
```

---

### 2. 二分查找**高频面试题**
1. **有序数组找目标值**（标准）
2. **搜索插入位置**（下界模板）
3. **在排序数组中查找元素第一个和最后一个位置**
4. **寻找旋转排序数组中的最小值**
5. **搜索旋转排序数组**（无重复/有重复）
6. **寻找峰值**（局部最大）
7. **x 的平方根**（整数二分）
8. **寻找重复数**（二分+抽屉原理）
9. **第k小的元素 / 矩阵中第k小**
10. **分割数组最大值最小**（经典二分答案）

---

## 二、二分排序（核心只有：**归并排序、快速排序**）
面试只考这两个，**原理+复杂度+手写+优缺点**必须全会。

### 1. 快速排序（最常考）
**思想**：分治  
- 选 pivot  
- 分区：< pivot 放左，≥ 放右  
- 递归左右

**复杂度**：
- 平均：**O(n log n)**
- 最坏：**O(n²)**（已排序+选首尾为pivot）
- 空间：递归栈 O(log n) ~ O(n)

**面试必问**：
- 如何优化最坏情况？**随机pivot / 三数取中**
- 快排是**不稳定**排序
- 快排 vs 归并：**原地、常数更小**

### 2. 归并排序
**思想**：分治  
- 递归拆分成两半  
- 分别排序  
- **合并两个有序数组**

**复杂度**：
- 最好/最坏/平均：**O(n log n)**
- 空间：**O(n)**

**面试必问**：
- 归并是**稳定**排序
- 用途：**链表排序最优解**（不用额外空间）、逆序对统计

---

## 三、二分**通用解题套路**（看到就用）
1. 数组**有序/部分有序** → 优先二分
2. 求**最小最大、最大最小、满足条件的边界** → **二分答案**
3. 旋转数组：**一定有一半有序**，用这个性质判断
4. 重复数/范围数：用**二分+计数**，不用哈希
5. 边界死循环：统一用 `left < right` 最稳

# 算法面试：二叉树**必考套路+遍历

二叉树面试**90% 就考三件事**：**遍历（递归+迭代）、深度/高度、路径问题**，再加几个经典变种。下面是**最精简、可直接背**的版本。

## 一、二叉树必掌握：4 种遍历（递归+迭代都要会）
### 1. 递归遍历（默写级）
- 前序：**根 → 左 → 右**
- 中序：**左 → 根 → 右**（BST 升序）
- 后序：**左 → 右 → 根**（最常考：树高度、子树依赖）
- 层序：**按层遍历**（BFS）

### 2. 迭代遍历（面试高频，防止不让写递归）
- 前序：栈，**先压右再压左**
- 中序：一路走左到底，出栈再右
- 后序：前序翻转（根→右→左 → 逆序）or 标记法
- 层序：**队列**，一层一处理（求宽度、右视图、层平均）

---

## 二、二叉树核心思想（所有题通用）
1. **后序遍历思想（最重要）**
   先处理**左子树**，再处理**右子树**，最后用结果**更新根**
   → 适用：求高度、直径、路径和、最大路径和、验证平衡树

2. **分治（Divide & Conquer）**
   树 = 根 + 左子树 + 右子树
   几乎所有树题都是：
   
> 左边算完 → 右边算完 → 合并结果
   
3. **BFS（层序）**
   求**宽度、层数、最近叶子、最底层最左节点、右视图**

---

## 三、面试**最高频题型**（按出现频率排序）
### 1. 求二叉树高度 / 最大深度
- 后序：`1 + max(left, right)`
- 变种：最小深度

### 2. 判断平衡二叉树
- 后序求高度，同时判断 `abs(left-right) > 1`
- 要求：**O(n)**，不能重复求高度

### 3. 二叉树直径（最长两节点距离）
- 后序：直径 = 左高 + 右高，全局更新最大值
- 返回给上一层：`1 + max(left, right)`

### 4. 翻转二叉树（经典题）
- 递归：交换左右子树
- 迭代：BFS/DFS 逐个交换

### 5. 路径总和（系列必考）
1. 是否存在**根到叶**路径和 = target
2. 找出**所有**根到叶路径
3. **任意路径**和 = target（路径可起止任意节点，只能向下）

### 6. 最大路径和（Hard 高频）
- 后序：
  - 每节点贡献：`val + max(left, right)`
  - 全局最大：`val + left + right`

### 7. 最近公共祖先 LCA
- 递归最简模板：
  - 找到 p/q 返回自身
  - 左右都有结果 → 返回当前节点
  - 否则返回有值的一边
- BST 特殊版：利用大小关系

### 8. 二叉树展开为链表（前序原地展开）

### 9. 层序系列
- 层平均值、右视图、锯齿遍历、填充每个节点的下一个右侧指针

### 10. 构造二叉树（必考）
- 前序 + 中序 → 构造二叉树
- 中序 + 后序 → 构造二叉树
- 思想：找根 → 拆分左右区间 → 递归构造

### 11. BST 二叉搜索树必考
- 中序 = 升序
- 验证 BST（递归传上下界）
- 第 k 小元素（中序计数）
- 增/删/查节点

### 12. 对称二叉树
- 双递归：左左对右右，左右对右左

---

## 四、二叉树解题万能套路（面试直接用）
1. **一看就会的简单题**：直接**后序**（高度、直径、平衡）
2. **要按层/最近/宽度**：直接**BFS 队列**
3. **BST 问题**：直接**中序遍历**
4. **找路径/子树依赖**：**后序 + 全局变量**
5. **构造树**：找根 + 分割区间 + 递归
6. **LCA**：标准递归四句搞定

---

> **遍历递归最基础，迭代栈队要熟练；**
> **高度直径平衡树，全靠后序来搞定；**
> **BST 必用中序升，构造二叉找根分；**
> **路径总和LCA，分治思想走天下。**

# 算法面试：**堆 / 栈 / 队列**

这三个结构**面试极其高频**，特点是：**题型固定、套路固定、代码模板固定**，背会直接秒杀。

## 一、栈 Stack（必考：单调栈 + 括号 + 递归非递归）
### 1. 核心考察点
- **先进后出**
- 天然适合：**成对匹配、回文、逆序、函数调用**

### 2. 三大必考题
#### （1）有效括号 / 最长有效括号
- 思路：栈存**下标**，遇到 `)` 匹配弹出，计算长度
- 经典：`()()`、`(())`、`())` 判断

#### （2）**单调栈**（最高频！）
- 用途：找**下一个更大/更小元素**、接雨水、柱状图最大矩形
- 套路：
  - 保持栈**单调递增/递减**
  - 不满足时弹出，计算答案
- 必背题：
  - 下一个更大元素 I/II
  - 接雨水（单调栈解法）
  - 柱状图中最大矩形

#### （3）二叉树遍历迭代版
- 前/中/后序 用栈模拟递归
- 中序：一路向左到底再出栈

---

## 二、队列 Queue（必考：BFS + 单调队列）
### 1. 普通队列
- **先进先出**
- 核心用途：**BFS 层序遍历**（二叉树、图、最短路径）

### 2. 高频题型
#### （1）二叉树层序遍历
- 队列 `levelSize` 控制一层

#### （2）图 / 网格 BFS
- 岛屿数量、腐烂橘子、迷宫最短路径
- 套路：队列 + 访问标记 + 方向数组

#### （3）**单调队列**（滑动窗口最大值）
- 维护**双端队列 Deque**，存下标，保证队内**递减**
- 队头就是当前窗口最大值

---

## 三、堆 Heap（优先队列，必考：TopK / 中位数）
### 1. 核心性质
- 大顶堆：堆顶最大
- 小顶堆：堆顶最小
- 插入/弹出：`O(log n)`

### 2. 三大必考场景
#### （1）TopK 问题
- 求**前 K 大**：用**小顶堆**，超过 K 就弹出最小
- 求**前 K 小**：用**大顶堆**

#### （2）数据流中位数
- 大顶堆存**左半小值**，小顶堆存**右半大值**
- 保证大顶堆.size = 小顶堆.size 或 +1
- 中位数直接取堆顶

#### （3）合并K个有序链表
- 小顶堆每次取最小节点

---

## 四、三者对比一句话
- **栈**：配对、逆序、**单调栈找更大更小**
- **队列**：BFS、层序、最短路径、**单调队列滑动窗口**

# 算法面试：**哈希（Hash）** 

哈希是面试**最实用、最无脑、最高频**的解法，**90% 题都是：空间换时间**。
核心就考：**Hash 思想 + 哈希表用法 + 高频题型**，背完直接秒题。

## 一、哈希本质（一句话）

- **Key → Hash 函数 → Index → Value**
- **O(1) 查、增、删**
- 面试等价于：**HashMap / HashSet**

## 二、两大核心工具（必选）

1. **HashSet**
   - 去重、判断**存在性**
2. **HashMap**
   - 记录**映射关系、计数、下标**

---

## 三、哈希**最常考 8 大套路**（直接背）

### 1. 两数之和（哈希鼻祖）

- 遍历一遍，**map 存【值→下标】**
- 检查 `target - nums[i]` 是否存在

### 2. 数组去重 / 判断重复元素

- 直接丢 **HashSet**，存在则重复

### 3. 字符/数字**频次统计**

- HashMap / 数组（字符用 `[128]`/`[26]`）
- 适用：异位词、出现次数、多数元素

### 4. 异位词（字母相同排列不同）

- 方法1：**排序后字符串做 key**（最简）
- 方法2：**字符计数数组转 String** 做 key（最优）
- 题目：字母异位词分组、有效异位词

### 5. 前缀和 + 哈希（子数组和问题）

- 公式：`preSum[i] - preSum[j] = k`
- 用 **HashMap 存前缀和出现次数**
- 题目：和为 K 的子数组、连续子数组和

### 6. 滑动窗口 + 哈希（最长无重复子串）

- HashMap 存**字符最后出现下标**
- 快速移动左指针，保证窗口无重复

### 7. 模拟/映射关系

- 罗马数字、链表随机节点、LRU 底层结构
- LRU 必考：**HashMap + 双向链表**

### 8. 哈希+堆组合（Top K 高频）

- 先用 HashMap 统计频次
- 再用**小顶堆**取 TopK

# 算法面试：**递归 & 回溯** 核心考察方法

递归+回溯是面试**中等偏上难度**的重灾区，但**套路极度固定**，只要掌握**通用模板 + 剪枝 + 题型分类**，基本全通吃。

## 一、先搞懂：递归 vs 回溯（一句话）
1. **递归**
   - 自己调用自己，**把大问题拆成小问题**
   - 二叉树、分治、动态规划都离不开
2. **回溯**
   - **递归的一种**，本质是**暴力搜索所有可能**
   - 走不通**回退一步**，换条路再试（**试错+撤销**）
   - 公式：**选择 → 递归 → 撤销（回溯）**

## 二、回溯万能模板（所有题都长这样）
```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        结果.add(路径)
        return
    
    for 选择 in 选择列表:
        # 1. 做选择
        路径.add(选择)
        # 2. 递归
        backtrack(路径, 新选择列表)
        # 3. 撤销选择（回溯）
        路径.remove(选择)
```
**核心三行：加 → 递归 → 删**

---

## 三、递归 高频考察点
### 1. 递归三大要素
1. **递归终止条件**（必写，否则栈溢出）
2. **递归递推式**（怎么拆小问题）
3. **返回值合并**（子结果怎么拼起来）

### 2. 高频递归场景
- 二叉树：前中后序、求高度、LCA、最大路径和
- 分治：归并排序、快速排序
- 链表：反转链表、合并有序链表
- 动态规划：自顶向下记忆化递归

### 3. 面试必问
- 递归 vs 迭代：优缺点、空间复杂度（栈深度）
- 尾递归：是否能优化栈空间
- 如何把递归改成迭代（栈模拟）

---

## 四、回溯 最高频 6 大类（必考）
### 1. 排列问题（Permutation）
- 全排列、含重复元素全排列
- **关键点**：**used 数组**标记是否用过
- 去重：排序 + 判断`nums[i] == nums[i-1] && !used[i-1]`

### 2. 组合问题（Combination）
- 组合总和、从n选k个
- **关键点**：**startIndex**（保证不回头选，避免重复）
- 不排列，只组合

### 3. 子集问题（Subset）
- 全部子集、含重复子集
- 每一步都要**保存当前路径**
- 去重：排序 + 同层跳过相同元素

### 4. 切割问题
- 分割回文串、复原IP地址
- 本质：在位置上**切一刀**，递归切剩下的

### 5. 棋盘/二维搜索
- N皇后、解数独
- 按行/列递归，判断合法性+剪枝

### 6. 图/矩阵搜索
- 单词搜索（word search）
- 上下左右四方向回溯，**标记走过 → 递归 → 取消标记**

---

## 五、回溯必背技巧（决定你能不能写对）
1. **去重必排序**
   组合/排列/子集有重复元素，**先排序**，再同层跳过重复
2. **排列用 used，组合用 start**
3. **剪枝（面试加分）**

# 算法面试：**双指针**

双指针是**最简单、最通用、写得最快**的算法思想，**数组、链表、字符串、二分**全靠它。
我给你整理成**最清晰、可直接背**的版本，覆盖所有面试题型。

---

## 一、双指针本质（一句话）
用**两个变量代替一层循环**，把 **O(n²) → O(n)**
核心就三类：
1. **左右指针**（一头一尾向中间）
2. **快慢指针**（一快一慢同向）
3. **滑动窗口**（左右边界形成窗口）

---

## 二、三大类双指针（所有题都归这3类）
### 1. 左右指针（左右两端 → 中间）
**最常考：反转、回文、两数之和、盛水最多**
- 典型用法：
  - `left=0，right=len-1`
  - 满足条件则**left++ / right--**

**高频题：**
- 验证回文字符串
- 反转字符串、反转元音字母
- **两数之和 II（有序数组）**
- **盛最多水的容器**（必考）
- 三数之和、四数之和（排序+左右指针）

---

### 2. 快慢指针（同向，一快一慢）
**最常考：链表、数组去重、移除元素**
- 典型用法：
  - 快指针探路，慢指针记录位置
  - 满足条件 `slow++`

**高频题：**
- **移除元素**（快探慢覆盖）
- **删除有序数组重复项**
- 链表：**找环、找入口、找中点、删除倒数第n**
- 移动零（把非零移到前面，剩下补零）

---

### 3. 滑动窗口（左右指针形成区间）
**字符串/子数组最高频**：最长、最短、满足条件子串
- 典型用法：
  - 右指针扩展，不满足则左指针收缩
  - 用哈希/数组计数

**高频题：**
- 最长无重复字符子串
- 最小覆盖子串
- 找到所有字母异位词
- 子数组和 ≤ K 等

---

## 三、面试最常考的**10道原题**（全是双指针）
1. 两数之和 II（有序）
2. 三数之和
3. 盛最多水的容器

5. 验证回文串

7. 移动零

10. 最小覆盖子串

> **链表环与中点找，快慢指针必出手；**
> **子串长短求极值，滑动窗口最拿手。**

# 算法面试：**动态规划 DP**

## 一、DP 核心三要素（必须先懂）
1. **dp 数组定义**（最重要）
   - `dp[i]` / `dp[i][j]` 到底代表什么？
2. **状态转移方程**
   - 怎么从**前面的状态**推到当前状态
3. **初始化 + 遍历顺序**
   - 从哪开始、从左/从右/从上到下？

一句话口诀：
> **定义dp → 找转移 → 做初始化 → 按序遍历**

---

## 二、DP 最常考 8 大类型（全覆盖）
### 1. 斐波那契类（入门）
- 爬楼梯、斐波那契、泰波那契
- 转移：`dp[i] = dp[i-1] + dp[i-2]`
- 可优化**空间O(1)**

### 2. 子序列问题（高频）
#### （1）最长递增子序列 LIS
- `dp[i] = max(dp[j] + 1)  j < i && nums[j] < nums[i]`
- O(n²) 必会，O(n log n) 贪心+二分加分

#### （2）最长公共子序列 LCS
- 两字符串
- `dp[i][j] = dp[i-1][j-1] + 1`（相等）
- `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`（不等）

### 3. 子数组/子串（连续）
- 最大子数组和（Kadane）
  - `dp[i] = max(nums[i], dp[i-1] + nums[i])`
- 最长重复子串、最长回文子串（DP版）

### 4. 背包问题（重中之重）
#### （1）01 背包（物品只能用一次）
- 模板：
  ```
  for i 1..n
    for j target..0
      dp[j] = max(dp[j], dp[j - w[i]] + v[i])
  ```
- 题目：分割等和子集、目标和、最后一块石头重量

#### （2）完全背包（物品无限）
- 模板：
  ```
  for i 1..n
    for j 0..target
      dp[j] = max(dp[j], dp[j - w[i]] + v[i])
  ```
- 题目：零钱兑换、组合总和IV

### 5. 打家劫舍系列（必背）
- 打家劫舍 I：线性
  - `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
- 打家劫舍 II：环形（拆成两个线性）
- 打家劫舍 III：二叉树树形 DP

### 6. 股票买卖系列（高频）
- 最多 1 次、无限次、最多 2 次、含冷冻期、含手续费
- 统一模板：**二维 DP**
  - `dp[i][0]`：第i天不持有的最大利润
  - `dp[i][1]`：第i天持有的最大利润

### 7. 路径问题
- 不同路径、不同路径II（障碍）
- 最小路径和
- 转移：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`

### 8. 区间 DP / 字符串编辑
- 编辑距离（经典Hard）
- 戳气球、最长回文子序列
- 一般按**区间长度从小到大**遍历

## 三、DP 通用解题步骤（面试直接说）
1. **确定 dp 含义**（一维/二维）
2. **找递推关系**（看前一步/前几步）
3. **初始化**（dp[0], dp[0][0]…）
4. **确定遍历顺序**（从小到大/从后往前/按区间）
5. **举例推导验证**
