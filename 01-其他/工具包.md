

## **Lombok 工具包**

### 一、Lombok 到底属于什么？

**官方/标准定义：**

1. **Java 编译时注解处理器（Annotation Processor）**
2. **字节码生成工具**
3. **Java 开发效率类库（工具jar）**

一句话：
**Lombok 是在编译阶段（.java → .class）帮你自动生成代码的工具，运行时不参与，不驻留进程。**

### 二、为什么**不是中间件**？

#### 中间件的特征（面试标准）：

- 独立运行或**驻留进程**
- 提供**独立服务**（Redis、MQ、MySQL、Nacos、ES、Dubbo、Minio等）
- 运行时依赖，**跨应用/跨进程**提供能力

#### Lombok 完全不符合：

- **只在编译时生效**，运行时不需要
- 不启动进程、不提供服务
- 只是**生成代码**，最终和普通 Java 代码一样运行在 JVM 里

### 三、面试标准答案（背这句）

> **Lombok 不属于中间件，它是 Java 编译时注解处理器，属于开发工具类库，仅在编译阶段生成 getter/setter/构造器等字节码，运行时无依赖、无进程、无服务。**

### 四、简单归类（帮你彻底分清）

- **框架**：Spring、SpringBoot、MyBatis
- **中间件**：Redis、RabbitMQ、MySQL、Nacos、Seata、Sentinel
- **工具包**：Lombok、HuTool、Guava、Fastjson
- **构建工具**：Maven、Gradle

---

如果你面试被问：
**“你们项目用了哪些中间件？”**
绝对不要说 Lombok！说了会被认为**概念不清**。

我可以帮你整理一份**面试绝对不会混淆的：框架 / 中间件 / 工具类 / 技术组件 清晰分类表**，要吗？



## Lombok注解

Lombok 是 Java 开发中广泛使用的注解库，它通过**编译期自动生成样板代码**（如 getter、setter、构造函数、toString 等），显著减少重复代码，提升开发效率。

以下是 **Lombok 最常用、最实用的注解**，按使用频率和场景分类整理：

------

### ✅ 1. `@Data` —— **万能注解（最常用）**

> 自动生成：  
>
> - 所有字段的 `getter` / `setter`  
> - `toString()`  
> - `equals()` 和 `hashCode()`  
> - 一个包含所有 **final 或 @NonNull 字段** 的构造函数（`@RequiredArgsConstructor`）

```java
@Data
public class User {
    private Long id;
    private String name;
}
```

> ⚠️ 注意：**不生成无参构造函数**！若需用于 MyBatis、Jackson 反序列化，要额外加 `@NoArgsConstructor`。

------

### ✅ 2. `@Getter` / `@Setter`

> 为类或字段生成 getter/setter，支持细粒度控制。

```java
@Getter
@Setter
public class Product {
    private String name;
    
    @Setter(AccessLevel.PROTECTED) // 限制 setter 访问级别
    private double price;
}
```

------

### ✅ 3. `@NoArgsConstructor` —— **无参构造函数**

> 生成无参构造器，**MyBatis、JPA、JSON 反序列化必备**。

```java
@Data
@NoArgsConstructor
public class OrderDTO {
    private String orderNo;
}
```

------

### ✅ 4. `@AllArgsConstructor` —— 全参构造函数

> 生成包含所有字段的构造函数。

```java
@AllArgsConstructor
public class ApiResponse {
    private int code;
    private String message;
}
```

------

### ✅ 5. `@Builder` —— 建造者模式

> 支持链式调用创建对象，适合字段较多的场景。

```java
@Builder
public class EmployeeLoginVO {
    private Long id;
    private String userName;
    private String token;
}

// 使用
EmployeeLoginVO vo = EmployeeLoginVO.builder()
    .id(1L)
    .userName("zhangsan")
    .token("abc")
    .build();
```

------

### ✅ 6. `@Slf4j` —— 自动注入日志对象（超常用！）

> 自动生成 SLF4J 日志实例，无需手写 `LoggerFactory.getLogger(...)`

```java
@Service
@Slf4j
public class PaymentService {
    public void process() {
        log.info("支付开始");
        log.error("支付失败", e);
    }
}
```

> 其他日志注解：`@Log4j2`、`@CommonsLog`，但 `@Slf4j` 是主流。

------

### ✅ 7. `@RequiredArgsConstructor` —— 构造函数注入（Spring 推荐）

> 为 **final 字段** 或 **@NonNull 字段** 生成构造函数，常用于 Spring 依赖注入。

```java
@Service
@RequiredArgsConstructor
public class UserService {
    private final UserRepository userRepo; // 自动注入
}
```

> ✅ Spring Boot 3+ 官方推荐替代 `@Autowired` 的方式。

------

### ✅ 8. `@ToString` —— 自定义 toString

> 可排除敏感字段（如密码）。

```java
@ToString(exclude = "password")
public class User {
    private String username;
    private String password; // 不会出现在 toString 中
}
```

------

### ✅ 9. `@EqualsAndHashCode` —— 自定义 equals/hashCode

> 指定参与比较的字段。

```java
@EqualsAndHashCode(of = "id")
public class Product {
    private Long id;
    private String name; // name 不参与 equals 判断
}
```

------

### ✅ 10. `@NonNull` —— 自动生成空值检查

> 在方法参数或字段上使用，自动插入非空校验。

```java
public void setName(@NonNull String name) {
    // 编译后自动插入: if (name == null) throw new NullPointerException("name is marked non-null");
    this.name = name;
}
```

------

### 📌 经典组合用法

| 场景                           | 推荐注解                                               |
| ------------------------------ | ------------------------------------------------------ |
| **DTO / VO / POJO**            | `@Data` + `@NoArgsConstructor`                         |
| **数据库实体（MyBatis/JPA）**  | `@Data` + `@NoArgsConstructor` + `@AllArgsConstructor` |
| **Spring Service（构造注入）** | `@RequiredArgsConstructor` + `@Slf4j`                  |
| **不可变对象**                 | `@Value`（相当于 `@Data` + 所有字段 `final`）          |

------

### ⚠️ 注意事项

1. **必须添加依赖**（Maven）：

   ```xml
   <dependency>
       <groupId>org.projectlombok</groupId>
       <artifactId>lombok</artifactId>
       <scope>provided</scope>
   </dependency>
   ```

2. **IDE 需安装插件**：

   - IntelliJ IDEA：安装 **Lombok 插件**，并启用 **Annotation Processing**
   - Eclipse：需运行 `lombok.jar` 安装

3. **避免在实体类中滥用 `@Data`**：
   Hibernate/JPA 实体若重写了 `equals/hashCode`，可能引发懒加载问题，建议只用 `@Getter`/`@Setter`。

------

### ✅ 总结：Top 5 必学注解

| 注解                       | 用途                                   |
| -------------------------- | -------------------------------------- |
| `@Data`                    | 一键生成 getter/setter/toString/equals |
| `@NoArgsConstructor`       | 提供无参构造（兼容框架）               |
| `@Slf4j`                   | 简化日志                               |
| `@Builder`                 | 链式创建复杂对象                       |
| `@RequiredArgsConstructor` | Spring 构造注入                        |

掌握这些，你的 Java 代码将更简洁、专业、高效！

## **Fastjson** 

**Fastjson** 是阿里巴巴开源的 **Java 高性能 JSON 序列化/反序列化工具库**，核心作用是在 **Java 对象 ↔ JSON 字符串** 之间做高效转换。**不属于中间件**，属于 **开发工具类库**（和 Lombok、Guava 同属一类）。

---

### 一、核心定位（面试必背）

- **全称**：Fastjson（com.alibaba:fastjson）
- **定位**：**JSON 解析 & 生成工具库**（序列化/反序列化）
- **开发者**：阿里巴巴
- **协议**：Apache 2.0 开源
- **版本**：
  - **1.x**：旧版（1.2.83 为最终安全版，已停止维护）
  - **2.x（Fastjson2）**：重构版，**官方推荐、安全、高性能**

### 二、核心特点（面试高频）

1. **极致性能**
   - 速度远超 Jackson、Gson，大数据量下优势明显
   - 底层用 **ASM 字节码生成** 减少反射开销
   - 内存占用低、GC 友好

2. **API 极简（静态方法）**
   ```java
   // 序列化：对象 → JSON
   String json = JSON.toJSONString(user);

   // 反序列化：JSON → 对象
   User user = JSON.parseObject(json, User.class);

   // 泛型集合
   List<User> list = JSON.parseArray(jsonArray, User.class);
   ```

3. **功能强大**
   - 支持 **JavaBean、List、Map、Date、Enum、泛型、嵌套对象**
   - **@JSONField**：别名、日期格式、序列化控制
   - 循环引用处理（`$ref`）
   - 自定义序列化器/反序列化器

4. **零依赖**
   
   - 不依赖第三方 Jar，仅 JDK 即可运行

### 三、核心 API（必须掌握）

#### 1. 序列化（对象 → JSON）

```java
// 基础
JSON.toJSONString(obj);

// 格式化输出
JSON.toJSONString(obj, SerializerFeature.PrettyFormat);

// 日期格式化
JSON.toJSONString(obj, SerializerFeature.WriteDateUseDateFormat);
```

#### 2. 反序列化（JSON → 对象）

```java
// 对象
User user = JSON.parseObject(json, User.class);

// List
List<User> list = JSON.parseArray(json, User.class);

// Map / 泛型（TypeReference）
Map<String, User> map = JSON.parseObject(json, new TypeReference<Map<String, User>>() {});
```

#### 3. 常用注解 @JSONField

```java
public class User {
    @JSONField(name = "user_id") // 别名
    private Long id;

    @JSONField(format = "yyyy-MM-dd") // 日期格式
    private Date createTime;

    @JSONField(serialize = false) // 不序列化
    private String password;
}
```

### 四、版本与安全（面试必问坑点）

#### 1. 1.x 严重安全问题（反序列化 RCE 漏洞）

- **≤1.2.24**：默认开启 AutoType，**可直接远程代码执行**
- **1.2.25~1.2.80**：黑名单机制，**可被绕过**
- **1.2.83+**：最终安全版（官方停止维护）

**结论**：**禁止使用 1.2.80 以下版本**

#### 2. Fastjson2（推荐）
- 彻底重构，**默认关闭 AutoType**，从根源解决安全问题
- 性能比 1.x 再提升 30%+
- 支持 Java 8+（LocalDateTime 等）
- 兼容 Spring Boot

### 五、Fastjson vs Jackson vs Gson（面试对比）
| 特性 | Fastjson | Jackson | Gson |
| :--- | :--- | :--- | :--- |
| **性能** | 最快 | 中 | 较慢 |
| **安全** | 1.x 高危；2.x 安全 | 安全 | 安全 |
| **Spring 默认** | ❌ | ✅ | ❌ |
| **API** | 极简 | 稍复杂 | 简洁 |
| **依赖** | 零依赖 | 需 jackson-databind | 无额外依赖 |

### 六、面试一句话总结
**Fastjson 是阿里开源的高性能 Java JSON 工具库，以速度快、API 简单著称，但 1.x 存在严重反序列化漏洞，新项目必须用 Fastjson2 或直接用 Spring 默认的 Jackson。**

## Log4j、Log4j2、SLF4J、Logback 

我现在**把 Log4j、Log4j2、SLF4J、Logback 四者一次性讲透**，这是**Java 日志体系最标准、面试官最爱问的完整版**，结构极度清晰，**直接背就能拿满分**。

### 一、先把四个东西彻底分清（最重要）

1. **SLF4J —— 日志门面（接口）**
2. **Logback —— 日志实现（SpringBoot 默认，官方推荐）**
3. **Log4j 1.x —— 老实现，已废弃、高危漏洞**
4. **Log4j 2 —— 新一代实现，性能极强，媲美 Logback**

---

### 二、Log4j2 是什么？

**Apache Log4j 2 = Log4j 的**完全重写升级版**。**
- 不是 Log4j 1.x 的小更新，是**全新架构**
- 解决了 Log4j 1.x 所有缺陷
- **高性能、异步、低锁、高吞吐**
- 支持：插件化、自定义、lambda、延迟日志、滚动压缩……

**地位：目前企业两大主流日志实现之一：Logback vs Log4j2**

---

### 三、核心区别：Log4j1 vs Logback vs Log4j2

| 框架          | 类型 | 状态               | 性能     | 特点                            |
| ------------- | ---- | ------------------ | -------- | ------------------------------- |
| **Log4j 1.x** | 实现 | **废弃、高危漏洞** | 低       | 老、慢、不维护                  |
| **Logback**   | 实现 | **主流、稳定**     | 高       | SpringBoot 默认、原生支持 SLF4J |
| **Log4j 2**   | 实现 | **现代、极高性能** | **极高** | 异步超强、无锁、多线程最优      |

一句话：
**Log4j2 是目前 Java 日志里性能最强的，但 Logback 最普及。**

---

### 四、Log4j2 与 SLF4J 的关系

Log4j2 **本身也提供门面（Log4j2 API）**，但**也完美支持 SLF4J**。

企业标准用法：
```
业务代码 → SLF4J → log4j-slf4j2-impl → Log4j2 实现
```

也就是：
**SLF4J 门面 + Log4j2 实现**，完全可行。

---

### 五、Log4j2 的核心优势（面试必说）

1. **完全无锁 / 低锁设计，高并发下极快**
2. **异步日志性能远超 Logback**
3. 支持 **lambda 懒加载日志**：
   ```java
   logger.info("数据：{}", () -> getUserData());
   ```
4. 支持 **自定义布局、过滤器、插件**
5. 配置支持 **XML/JSON/YAML**
6. **自动重载配置**，不用重启
7. 修复了 Log4j1 的所有漏洞（注意：Log4j2 也出过 Log4Shell，但已修复）

---

### 六、最重要结论（面试必背）

1. **SLF4J = 门面（接口），必须用**
2. **Logback = 最稳定、最常用、SpringBoot 默认**
3. **Log4j2 = 性能最强、高并发首选**
4. **Log4j 1.x = 废弃、禁止使用**

**主流组合：**
- **SLF4J + Logback**（90% 公司）
- **SLF4J + Log4j2**（高并发、高性能项目）

---

### 七、超精简一句话总结（面试官最爱）

- **SLF4J 是日志接口，统一规范。**
- **Logback 是稳定主流实现，SpringBoot 默认。**
- **Log4j1 已废弃。**
- **Log4j2 是新一代高性能实现，高并发最强。**

### @Slf4j 注解（Java 日志开发终极版）

**@Slf4j** 是目前 Java 开发中**最主流、最推荐**的日志注解，主要用于 **Lombok 框架**。

它的核心作用是：**帮你自动生成日志对象代码，省去手动写 `Logger` 实例的麻烦。**

---

#### 一、核心作用：为什么要用 @Slf4j？

##### 1. 极简代码（核心卖点）

**不用写** 这种啰嗦代码：
```java
// 以前：每个类都要写这行
private static final Logger log = LoggerFactory.getLogger(UserService.class);
```

**只用写** 注解：
```java
@Slf4j
@Service
public class UserService {
    // 直接使用 log 对象，无需声明
    public void getUser() {
        log.info("获取用户信息..."); 
    }
}
```

##### 2. 编译时生成

Lombok 会在**编译阶段**，帮你把上面的代码自动生成进去。
- 运行时：和手动写 `Logger` 完全一样，没有任何性能损耗。
- 开发时：代码极度清爽。

---

#### 二、核心依赖（必须引入）

想使用 `@Slf4j`，必须引入 **Lombok** 依赖。

##### Maven

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <optional>true</optional>
</dependency>
```

##### Gradle

```groovy
compileOnly 'org.projectlombok:lombok'
annotationProcessor 'org.projectlombok:lombok'
```

---

#### 三、核心用法（面试/实战必知）

##### 1. 基本使用

直接加在类上，即可使用 `log` 变量：
```java
@Slf4j
public class OrderService {
    public void pay() {
        // 1. 普通日志
        log.info("支付开始");
        
        // 2. 占位符语法（推荐，性能优于字符串拼接）
        String orderNo = "10086";
        log.error("支付失败，订单号：{}", orderNo); 
        
        // 3. 异常打印（第二个参数传 Throwable）
        try {
            // ...
        } catch (Exception e) {
            log.error("支付异常", e); // 不要 log.error(e.getMessage(), e)
        }
    }
}
```

##### 2. 自定义日志对象名称

默认生成的变量名是 `log`。
如果你想自定义名字（比如叫 `logger`），可以使用 `@Slf4j(topic = "logger")`：
```java
@Slf4j(topic = "OrderLogger")
public class OrderService {
    // 此时使用的是 OrderLogger
    public void test() {
        OrderLogger.info("测试");
    }
}
```

---

#### 四、进阶技巧（高阶开发）

##### 1. 条件日志（避免字符串拼接损耗）

如果日志级别是 `debug`，通常不需要拼接字符串。Lombok 支持 `if` 逻辑：
```java
// 推荐写法
if (log.isDebugEnabled()) {
    log.debug("复杂对象：{}", expensiveObject.toString());
}
```
*注：现代 IDE 和 Logback/Log4j2 会自动优化占位符，其实可以直接写 `log.debug("...")`，但加判断是最稳妥的习惯。*

##### 2. 结合 SLF4J 2.x

现在的项目多用 **SLF4J 2.x + Logback/Log4j2**，`@Slf4j` 完全兼容。

---

#### 五、面试必问：@Slf4j 与 传统 Logger 的区别

| 特性           | @Slf4j (Lombok)        | 传统手动 Logger                  |
| :------------- | :--------------------- | :------------------------------- |
| **代码量**     | **极少**，一个注解搞定 | **多**，每类都要写 static Logger |
| **运行时性能** | **一致**（编译时生成） | 一致                             |
| **可维护性**   | **高**，统一规范       | 低，容易写错类名                 |
| **IDEA 支持**  | **完美**（安装插件）   | 完美                             |
| **替代方案**   | —                      | 无                               |

**一句话总结**：
**@Slf4j 是 Lombok 提供的语法糖，它在编译期生成标准的 SLF4J Logger 代码，是现代 Java 后端开发的标准配置。**

---
