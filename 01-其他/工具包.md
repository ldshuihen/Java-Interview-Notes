

## **Lombok 工具包**

### 一、Lombok 到底属于什么？

**官方/标准定义：**

1. **Java 编译时注解处理器（Annotation Processor）**
2. **字节码生成工具**
3. **Java 开发效率类库（工具jar）**

一句话：
**Lombok 是在编译阶段（.java → .class）帮你自动生成代码的工具，运行时不参与，不驻留进程。**

### 二、为什么**不是中间件**？

#### 中间件的特征（面试标准）：

- 独立运行或**驻留进程**
- 提供**独立服务**（Redis、MQ、MySQL、Nacos、ES、Dubbo、Minio等）
- 运行时依赖，**跨应用/跨进程**提供能力

#### Lombok 完全不符合：

- **只在编译时生效**，运行时不需要
- 不启动进程、不提供服务
- 只是**生成代码**，最终和普通 Java 代码一样运行在 JVM 里

### 三、面试标准答案（背这句）

> **Lombok 不属于中间件，它是 Java 编译时注解处理器，属于开发工具类库，仅在编译阶段生成 getter/setter/构造器等字节码，运行时无依赖、无进程、无服务。**

### 四、简单归类（帮你彻底分清）

- **框架**：Spring、SpringBoot、MyBatis
- **中间件**：Redis、RabbitMQ、MySQL、Nacos、Seata、Sentinel
- **工具包**：Lombok、HuTool、Guava、Fastjson
- **构建工具**：Maven、Gradle

---

**“你们项目用了哪些中间件？”**

## Lombok注解

Lombok 是 Java 开发中广泛使用的注解库，它通过**编译期自动生成样板代码**（如 getter、setter、构造函数、toString 等），显著减少重复代码，提升开发效率。

以下是 **Lombok 最常用、最实用的注解**，按使用频率和场景分类整理：

------

### ✅ 1. `@Data` —— **万能注解（最常用）**

> 自动生成：  
>
> - 所有字段的 `getter` / `setter`  
> - `toString()`  
> - `equals()` 和 `hashCode()`  
> - 一个包含所有 **final 或 @NonNull 字段** 的构造函数（`@RequiredArgsConstructor`）

```java
@Data
public class User {
    private Long id;
    private String name;
}
```

> ⚠️ 注意：**不生成无参构造函数**！若需用于 MyBatis、Jackson 反序列化，要额外加 `@NoArgsConstructor`。

------

### ✅ 2. `@Getter` / `@Setter`

> 为类或字段生成 getter/setter，支持细粒度控制。

```java
@Getter
@Setter
public class Product {
    private String name;
    
    @Setter(AccessLevel.PROTECTED) // 限制 setter 访问级别
    private double price;
}
```

------

### ✅ 3. `@NoArgsConstructor` —— **无参构造函数**

> 生成无参构造器，**MyBatis、JPA、JSON 反序列化必备**。

```java
@Data
@NoArgsConstructor
public class OrderDTO {
    private String orderNo;
}
```

------

### ✅ 4. `@AllArgsConstructor` —— 全参构造函数

> 生成包含所有字段的构造函数。

```java
@AllArgsConstructor
public class ApiResponse {
    private int code;
    private String message;
}
```

------

### ✅ 5. `@Builder` —— 建造者模式

> 支持链式调用创建对象，适合字段较多的场景。

```java
@Builder
public class EmployeeLoginVO {
    private Long id;
    private String userName;
    private String token;
}

// 使用
EmployeeLoginVO vo = EmployeeLoginVO.builder()
    .id(1L)
    .userName("zhangsan")
    .token("abc")
    .build();
```

------

### ✅ 6. `@Slf4j` —— 自动注入日志对象（超常用！）

> 自动生成 SLF4J 日志实例，无需手写 `LoggerFactory.getLogger(...)`

```java
@Service
@Slf4j
public class PaymentService {
    public void process() {
        log.info("支付开始");
        log.error("支付失败", e);
    }
}
```

> 其他日志注解：`@Log4j2`、`@CommonsLog`，但 `@Slf4j` 是主流。

------

### ✅ 7. `@RequiredArgsConstructor` —— 构造函数注入（Spring 推荐）

> 为 **final 字段** 或 **@NonNull 字段** 生成构造函数，常用于 Spring 依赖注入。

```java
@Service
@RequiredArgsConstructor
public class UserService {
    private final UserRepository userRepo; // 自动注入
}
```

> ✅ Spring Boot 3+ 官方推荐替代 `@Autowired` 的方式。

------

### ✅ 8. `@ToString` —— 自定义 toString

> 可排除敏感字段（如密码）。

```java
@ToString(exclude = "password")
public class User {
    private String username;
    private String password; // 不会出现在 toString 中
}
```

------

### ✅ 9. `@EqualsAndHashCode` —— 自定义 equals/hashCode

> 指定参与比较的字段。

```java
@EqualsAndHashCode(of = "id")
public class Product {
    private Long id;
    private String name; // name 不参与 equals 判断
}
```

------

### ✅ 10. `@NonNull` —— 自动生成空值检查

> 在方法参数或字段上使用，自动插入非空校验。

```java
public void setName(@NonNull String name) {
    // 编译后自动插入: if (name == null) throw new NullPointerException("name is marked non-null");
    this.name = name;
}
```

------

### 📌 经典组合用法

| 场景                           | 推荐注解                                               |
| ------------------------------ | ------------------------------------------------------ |
| **DTO / VO / POJO**            | `@Data` + `@NoArgsConstructor`                         |
| **数据库实体（MyBatis/JPA）**  | `@Data` + `@NoArgsConstructor` + `@AllArgsConstructor` |
| **Spring Service（构造注入）** | `@RequiredArgsConstructor` + `@Slf4j`                  |
| **不可变对象**                 | `@Value`（相当于 `@Data` + 所有字段 `final`）          |

------

### ⚠️ 注意事项

1. **必须添加依赖**（Maven）：

   ```xml
   <dependency>
       <groupId>org.projectlombok</groupId>
       <artifactId>lombok</artifactId>
       <scope>provided</scope>
   </dependency>
   ```

2. **IDE 需安装插件**：

   - IntelliJ IDEA：安装 **Lombok 插件**，并启用 **Annotation Processing**
   - Eclipse：需运行 `lombok.jar` 安装

3. **避免在实体类中滥用 `@Data`**：
   Hibernate/JPA 实体若重写了 `equals/hashCode`，可能引发懒加载问题，建议只用 `@Getter`/`@Setter`。

------

### ✅ 总结：Top 5 必学注解

| 注解                       | 用途                                   |
| -------------------------- | -------------------------------------- |
| `@Data`                    | 一键生成 getter/setter/toString/equals |
| `@NoArgsConstructor`       | 提供无参构造（兼容框架）               |
| `@Slf4j`                   | 简化日志                               |
| `@Builder`                 | 链式创建复杂对象                       |
| `@RequiredArgsConstructor` | Spring 构造注入                        |

掌握这些，你的 Java 代码将更简洁、专业、高效！

## JSON库

### Jackson / Fastjson2 / Gson

三大JSON库 适用场景+选型对照表

结合**后端企业开发、面试考点、实际落地**，一次性讲清，好记、好用、面试直接背。

#### 一、核心定位一句话

1. **Jackson**：Spring 官方默认，**企业级标准、稳、功能最强**
2. **Fastjson2**：阿里出品，**性能天花板、高并发首选**，修复所有安全漏洞
3. **Gson**：Google 轻量简洁，**Android 标配、小型项目、低配置**

---

#### 二、全方位横向对比

| 对比维度        | Jackson                  | Fastjson2                     | Gson                               |
| --------------- | ------------------------ | ----------------------------- | ---------------------------------- |
| 开发维护        | Spring 持续维护          | 阿里全新重构                  | Google 稳定维护                    |
| SpringBoot      | **原生内置，无依赖**     | 需手动引入                    | 需手动引入                         |
| 性能            | 优秀、均衡               | **最强，序列化/反序列化最快** | 中等、日常够用                     |
| 安全性          | 极高，无历史漏洞         | 高，默认关闭AutoType          | 极高，几乎无漏洞                   |
| 注解丰富度      | 最全、定制能力拉满       | 中等                          | 精简够用                           |
| 容错性          | 严格（多余字段默认报错） | 高容错                        | **默认最强容错，自动忽略多余字段** |
| 日期/时间       | 需要手动注册JSR310       | 原生友好                      | 需要手动配置                       |
| 大文件/流式解析 | 完美支持                 | 支持                          | 较弱                               |
| 定制扩展        | 极强（序列化器、过滤器） | 强                            | 弱                                 |
| 体积&轻量       | 中等                     | 中等                          | **最轻量**                         |
| 跨端场景        | 后端专属                 | 后端专属                      | **后端 + Android 双端**            |

---

#### 三、各自 专属适用场景（重点⭐）

##### 1. Jackson 适用场景

✅ 必选场景
- **所有 SpringBoot / SpringCloud 微服务项目**（默认集成，无需额外引入）
- 大型企业级项目、复杂业务、强规范项目
- 需要**高度自定义序列化**（脱敏、字典翻译、日期统一、Long转字符串）
- 金融、政务、政企系统：**稳定优先、安全优先**
- 依赖统一 Web 序列化、全局统一返回格式

❌ 不适合
- 极致追求极简、不想任何配置的小项目
- 超高QPS、百万级序列化高并发场景

---

##### 2. Fastjson2 适用场景

✅ 必选场景
- **高并发、大数据量接口**（日志、批量数据、导出、风控）
- 阿里生态项目、中间件、消息队列序列化
- 需要 JSONB 二进制序列化、追求极致吞吐
- 历史 Fastjson1 老项目**安全升级改造**
- 高性能RPC、缓存序列化场景

❌ 不适合
- 极简小项目（没必要）
- 强约束、强规范的传统政企系统

---

##### 3. Gson 适用场景

✅ 必选场景
- **Android 客户端开发（行业标配）**
- 小型后端项目、内部工具项目、脚本项目
- 快速开发、零配置、不想处理各种序列化异常
- 第三方简单对接、结构不稳定的JSON解析
- 轻量化应用、资源受限环境

❌ 不适合
- 大型微服务、复杂定制序列化
- 后端高并发接口

---

#### 四、结合你之前的「对象拷贝方案」联动选型

1. **业务常规拷贝 + 项目统一JSON**
   Spring项目 → 用 **Jackson 做深拷贝**，统一技术栈
2. **高并发批量DTO转换、大数据列表**
   → 选 **Fastjson2**，序列化速度更快
3. **简单项目、工具类、临时拷贝、客户端**
   → 选 **Gson**，代码极简、不易报错

---

#### 五、面试高频标准答案（直接背诵）

1. **Jackson**
SpringBoot 默认 JSON 框架，生态完善、扩展能力强、稳定性高，适合企业级微服务、复杂业务系统，是后端主流首选。

2. **Fastjson2**
阿里重构升级版，解决1.x安全漏洞，性能顶尖，适合高并发、大数据量、高性能服务场景。

3. **Gson**
Google 轻量JSON库，API简洁、容错性好，是Android标配，适合小型项目、客户端、轻量后端服务。

---

#### 六、最终最简选型口诀

- 微服务、Spring项目、稳第一 ➜ **Jackson**
- 高并发、大数据、性能第一 ➜ **Fastjson2**
- 安卓端、小项目、省事第一 ➜ **Gson**

### Fastjson 

Fastjson 是阿里巴巴开源的**高性能 Java JSON 库**，以**速度极快、使用简单、功能强大**著称，广泛用于对象序列化/反序列化、API 接口、缓存存储等场景。目前分为 **Fastjson 1.x**（经典版）和 **Fastjson 2**（重构安全版），**企业新项目强烈推荐 Fastjson 2**。

---

#### 一、核心定位与优势

##### 1. 为什么快？（底层原理）

- **ASM 字节码生成**：首次调用时动态生成 **get/set 代码**，**无反射开销**，性能接近手写
- **零拷贝、char[] 优化**：内部用字符数组，减少 String 对象创建
- **快速路径解析**：int/long/String 等基础类型专用解析逻辑
- **ThreadLocal 缓存**：复用解析器/序列化器，降低 GC

##### 2. 两大版本对比（关键）

| 维度   | Fastjson 1.x           | Fastjson 2                            |
| ------ | ---------------------- | ------------------------------------- |
| 包名   | `com.alibaba.fastjson` | `com.alibaba.fastjson2`               |
| 安全   | 历史漏洞多，需严格升级 | **默认安全**，AutoType 关闭           |
| 性能   | 快                     | **比 1.x 快 3～10 倍**                |
| 新特性 | 停止新功能             | JSONB、JSONPath、JSON Schema、GraalVM |
| JDK    | 8+                     | 8+/11+/17+ 深度优化                   |

**企业建议**：
- 新项目：直接用 **Fastjson 2**
- 老项目：1.x 必须升级到 **1.2.83+**（最终安全版）

---

    <artifactId>fastjson2</artifactId>

#### 三、核心 API（100% 覆盖业务）

##### 1. 序列化：Java 对象 → JSON 字符串

```java
import com.alibaba.fastjson2.JSON;

// 对象
User user = new User(1L, "张三", 20);
String json = JSON.toJSONString(user);

// 格式化输出（调试用）
String prettyJson = JSON.toJSONString(user, JSONWriter.Feature.PrettyFormat);

// 包含 null 值（默认不输出 null）
String jsonWithNull = JSON.toJSONString(user, JSONWriter.Feature.WriteNulls);
```

##### 2. 反序列化：JSON → Java 对象

```java
// 1. 转普通对象
User user = JSON.parseObject(json, User.class);

// 2. 转泛型（List/Map，解决泛型擦除）
List<User> userList = JSON.parseObject(jsonArray, new TypeReference<List<User>>() {});
Map<String, User> userMap = JSON.parseObject(jsonMap, new TypeReference<Map<String, User>>() {});

// 3. 转 JSONArray（动态解析）
JSONArray array = JSON.parseArray(jsonArray);
Long id = array.getJSONObject(0).getLong("id");
```

##### 3. 对象 ↔ Map 互转

```java
// 对象 → Map
Map<String, Object> map = JSON.parseObject(JSON.toJSONString(user), Map.class);

// Map → 对象
User user = JSON.parseObject(JSON.toJSONString(map), User.class);
```

#### 四、高级特性（企业必备）

##### 1. @JSONField 注解（字段定制）

```java
@Data
public class User {
    private Long id;

    // 1. 改名映射
    @JSONField(name = "user_name")
    private String username;

    // 2. 日期格式化
    @JSONField(format = "yyyy-MM-dd HH:mm:ss")
    private Date createTime;

    // 3. 忽略字段
    @JSONField(serialize = false, deserialize = false)
    private String password;

    // 4. ordinal 排序输出
    @JSONField(ordinal = 1)
    private Integer age;
}
```

##### 2. 日期/枚举/类型转换

- **全局日期格式**
```java
// Fastjson 2
JSON.config(JSONReader.Feature.UseDefaultDateFormatForDate);
JSON.config(JSONWriter.Feature.WriteDateFormat);
JSON.config("yyyy-MM-dd HH:mm:ss");
```

- **枚举处理**
```java
// 序列化枚举为 name() 或 ordinal
@JSONField(serializeUsing = EnumOrdinalSerializer.class)
private StatusEnum status;
```

##### 3. JSONPath（复杂提取）

```java
String json = "{\"user\":{\"name\":\"张三\",\"age\":20,\"tags\":[\"a\",\"b\"]}}";

// 1. 编译（复用，性能高）
JSONPath path = JSONPath.of("$.user.name");
String name = path.extract(json); // 张三

// 2. 数组
JSONPath tagsPath = JSONPath.of("$.user.tags[0]");
String tag = tagsPath.extract(json); // a
```

##### 4. JSONB（二进制协议，性能爆炸）

- 比 JSON 快 **3～9 倍**、体积小 **40%**，适合 RPC/缓存
```java
// 序列化
byte[] bytes = JSONB.toBytes(user);

// 反序列化
User user = JSONB.parseObject(bytes, User.class);
```

##### 5. BeanToArray（极简数组格式）

```java
// 输出：[1,"张三",20] 而非 {"id":1,"name":"张三","age":20}
String json = JSON.toJSONString(user, JSONWriter.Feature.BeanToArray);

// 反序列化
User user = JSON.parseObject(json, User.class, JSONReader.Feature.SupportArrayToBean);
```

#### 五、Spring Boot 集成（替换默认 Jackon）

```java
@Configuration
public class FastjsonConfig {
    @Bean
    public HttpMessageConverters fastjsonHttpMessageConverters() {
        FastJsonHttpMessageConverter converter = new FastJsonHttpMessageConverter();
        
        FastJsonConfig config = new FastJsonConfig();
        config.setDateFormat("yyyy-MM-dd HH:mm:ss");
        config.setWriterFeatures(
            JSONWriter.Feature.WriteNulls,
            JSONWriter.Feature.PrettyFormat
        );
        
        converter.setFastJsonConfig(config);
        converter.setSupportedMediaTypes(Collections.singletonList(MediaType.APPLICATION_JSON));
        
        return new HttpMessageConverters(converter);
    }
}
```

#### 六、安全问题（重中之重）

##### 1. 核心风险：AutoType 反序列化漏洞

- 原理：JSON 带 `@type` 类名，Fastjson 自动实例化，可能执行恶意代码
- **Fastjson 1.x**：默认关闭，但历史可绕过，**必须 1.2.83+**
- **Fastjson 2.x**：**默认完全禁用 AutoType**，安全基线极高

##### 2. 安全加固（必做）

1. **版本强制**
   - 1.x → **1.2.83+**
   - 2.x → **2.0.29+**
2. **禁用 AutoType（1.x）**
```java
// 全局关闭
ParserConfig.getGlobalInstance().setAutoTypeSupport(false);
// 开启 SafeMode（彻底禁止）
System.setProperty("fastjson.parser.safeMode", "true");
```
3. **白名单（仅必要时）**
```java
ParserConfig.getGlobalInstance().addAccept("com.xxx.entity");
```

---

#### 七、对象属性拷贝（和 MapStruct/Hutool 对比）

Fastjson 可做**临时拷贝**，适合简单场景：
```java
// 浅拷贝
UserDTO dto = JSON.parseObject(JSON.toJSONString(userDO), UserDTO.class);

// 深拷贝（推荐）
UserDTO dto = JSON.parseObject(JSON.toJSONString(userDO), UserDTO.class);
```

##### 方案对比（回到你之前的问题）

| 方案                | 性能  | 易用  | 不同名 | 深拷贝 | 适用场景              |
| ------------------- | ----- | ----- | ------ | ------ | --------------------- |
| **MapStruct**       | ⭐⭐⭐⭐⭐ | ⭐⭐⭐   | ✅      | ❌      | 企业级、高性能、稳定  |
| **Hutool BeanUtil** | ⭐⭐⭐   | ⭐⭐⭐⭐⭐ | ✅      | ✅      | 快速开发、中小项目    |
| **Fastjson**        | ⭐⭐⭐⭐  | ⭐⭐⭐   | ❌      | ✅      | 临时转换、JSON 同时用 |

**结论**：
- 纯属性拷贝：优先 **MapStruct**（性能）或 **Hutool**（简单）
- 同时要 JSON 处理：**Fastjson 2** 一库多用

---

#### 八、最佳实践总结

1. **新项目用 Fastjson 2**，安全 + 性能拉满
2. **核心 API**：`JSON.toJSONString()` / `JSON.parseObject()`
3. **泛型必用** `TypeReference`
4. **高频场景**用 **JSONB** 或 **BeanToArray**
5. **安全底线**：1.2.83+ / 2.0.29+，关闭 AutoType

### **Jackson** 

**Jackson** 是 **Spring Boot 默认的 JSON 处理库**，由 FasterXML 开源，以**稳定性高、扩展性极强、安全性好**著称，是**企业级 Java 项目的事实标准**。

---

#### 一、核心定位与优势

##### 1. 为什么选 Jackson？

- **Spring 全家桶默认**：无需额外引入，开箱即用
- **模块化设计**：核心 + 扩展模块（Java 8 时间、Guava、Kotlin 等）
- **三种处理模式**：**流式 API**（高性能）、**树模型**（灵活）、**数据绑定**（简单）
- **高度可定制**：注解、全局配置、自定义序列化器
- **内存高效**：处理大文件、复杂嵌套时 GC 友好
- **生态完善**：几乎支持所有数据类型与框架集成

##### 2. 三大核心模块（只需引入一个）

- **jackson-core**：底层流式 API（`JsonParser`/`JsonGenerator`）
- **jackson-annotations**：注解支持（`@JsonProperty` 等）
- **jackson-databind**：**数据绑定**（`ObjectMapper`，最常用）

```xml
<!-- Maven 依赖（Spring Boot 已内置） -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.15.3</version> <!-- 最新稳定版 -->
</dependency>
```

#### 二、核心 API：ObjectMapper（万能中枢）

`ObjectMapper` 是 Jackson 的**核心入口**，**线程安全**（全局单例即可）。

##### 1. 序列化：对象 → JSON

```java
import com.fasterxml.jackson.databind.ObjectMapper;

// 全局单例（推荐）
private static final ObjectMapper MAPPER = new ObjectMapper();

// 1. 基础序列化
User user = new User(1L, "张三", new Date());
String json = MAPPER.writeValueAsString(user);
// 输出：{"id":1,"username":"张三","createTime":1745678901234}

// 2. 格式化输出（调试）
String prettyJson = MAPPER.writerWithDefaultPrettyPrinter()
                          .writeValueAsString(user);

// 3. 忽略 null 值
MAPPER.setSerializationInclusion(JsonInclude.Include.NON_NULL);
```

##### 2. 反序列化：JSON → 对象

```java
String json = "{\"id\":1,\"username\":\"张三\"}";

// 1. 转普通对象
User user = MAPPER.readValue(json, User.class);

// 2. 转泛型（List/Map，解决泛型擦除）
// List<User>
List<User> userList = MAPPER.readValue(jsonArray, 
    new TypeReference<List<User>>() {});

// Map<String, User>
Map<String, User> userMap = MAPPER.readValue(jsonMap, 
    new TypeReference<Map<String, User>>() {});

// 3. 忽略未知字段（JSON 多字段不报错）
MAPPER.disable(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES);
```

##### 3. 树模型：JsonNode（动态解析）

适合 JSON 结构不确定的场景：
```java
// JSON → JsonNode
JsonNode node = MAPPER.readTree(json);
Long id = node.get("id").asLong();
String name = node.get("username").asText();

// JsonNode → JSON
String json = MAPPER.writeValueAsString(node);
```

#### 三、常用注解（字段级定制）

##### 1. @JsonProperty：字段重命名

```java
@Data
public class User {
    private Long id;

    // JSON 字段名为 user_name
    @JsonProperty("user_name")
    private String username;
}
```

##### 2. @JsonFormat：日期/格式控制

```java
// 日期格式化
@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
private Date createTime;

// 数字格式化
@JsonFormat(shape = JsonFormat.Shape.STRING)
private Long id; // 输出为字符串 "1" 而非数字 1
```

##### 3. @JsonIgnore：忽略字段

```java
@JsonIgnore // 不序列化、不反序列化
private String password;
```

##### 4. @JsonInclude：条件包含

```java
// 类级别：非 null 才序列化
@JsonInclude(JsonInclude.Include.NON_NULL)
public class User {}
```

##### 5. @JsonAlias：反序列化别名

```java
// 兼容 JSON 中的 name/userName/用户名 三种字段
@JsonAlias({"name", "userName", "用户名"})
private String username;
```

    }

#### 四、核心常用 API

##### 1. 序列化：对象 → JSON 字符串

```java
User user = new User(1L, "张三", "123456");

// 普通压缩JSON
String json = gson.toJson(user);

// 格式化美观JSON
String prettyJson = new GsonBuilder().setPrettyPrinting().create().toJson(user);
```

##### 2. 反序列化：JSON → 普通对象

```java
String json = "{\"id\":1,\"name\":\"张三\"}";
User user = gson.fromJson(json, User.class);
```

##### 3. 反序列化：泛型 List / Map

解决泛型擦除，使用 `TypeToken`
```java
// List<User>
String listJson = "[{\"id\":1},{\"id\":2}]";
List<User> userList = gson.fromJson(listJson, new TypeToken<List<User>>(){}.getType());

// Map
Map<String,Object> map = gson.fromJson(json, new TypeToken<Map<String,Object>>(){}.getType());
```

##### 4. 树模型 动态解析（JsonObject / JsonArray）

```java
JsonObject jsonObj = gson.fromJson(json, JsonObject.class);
Long id = jsonObj.get("id").getAsLong();
String name = jsonObj.get("name").getAsString();

JsonArray jsonArray = jsonObj.getAsJsonArray("list");
```

### @Slf4j 注解（Java 日志开发终极版）

**@Slf4j** 是目前 Java 开发中**最主流、最推荐**的日志注解，主要用于 **Lombok 框架**。

它的核心作用是：**帮你自动生成日志对象代码，省去手动写 `Logger` 实例的麻烦。**

---

#### 一、核心作用：为什么要用 @Slf4j？

##### 1. 极简代码（核心卖点）

**不用写** 这种啰嗦代码：
```java
// 以前：每个类都要写这行
private static final Logger log = LoggerFactory.getLogger(UserService.class);
```

**只用写** 注解：
```java
@Slf4j
@Service
public class UserService {
    // 直接使用 log 对象，无需声明
    public void getUser() {
        log.info("获取用户信息..."); 
    }
}
```

##### 2. 编译时生成

Lombok 会在**编译阶段**，帮你把上面的代码自动生成进去。
- 运行时：和手动写 `Logger` 完全一样，没有任何性能损耗。
- 开发时：代码极度清爽。

---

#### 二、核心依赖（必须引入）

想使用 `@Slf4j`，必须引入 **Lombok** 依赖。

##### Maven

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <optional>true</optional>
</dependency>
```

##### Gradle

```groovy

```

#### 三、核心用法（面试/实战必知）

##### 1. 基本使用

直接加在类上，即可使用 `log` 变量：
```java
@Slf4j
public class OrderService {
    public void pay() {
        // 1. 普通日志
        log.info("支付开始");
        
        // 2. 占位符语法（推荐，性能优于字符串拼接）
        String orderNo = "10086";
        log.error("支付失败，订单号：{}", orderNo); 
        
        // 3. 异常打印（第二个参数传 Throwable）
        try {
            // ...
        } catch (Exception e) {
            log.error("支付异常", e); // 不要 log.error(e.getMessage(), e)
        }
    }
}
```

##### 2. 自定义日志对象名称

默认生成的变量名是 `log`。
如果你想自定义名字（比如叫 `logger`），可以使用 `@Slf4j(topic = "logger")`：
```java
@Slf4j(topic = "OrderLogger")
public class OrderService {
    // 此时使用的是 OrderLogger
    public void test() {
        OrderLogger.info("测试");
    }
}
```

#### 四、进阶技巧（高阶开发）

##### 1. 条件日志（避免字符串拼接损耗）

如果日志级别是 `debug`，通常不需要拼接字符串。Lombok 支持 `if` 逻辑：
```java
// 推荐写法
if (log.isDebugEnabled()) {
    log.debug("复杂对象：{}", expensiveObject.toString());
}
```
*注：现代 IDE 和 Logback/Log4j2 会自动优化占位符，其实可以直接写 `log.debug("...")`，但加判断是最稳妥的习惯。*

##### 2. 结合 SLF4J 2.x

现在的项目多用 **SLF4J 2.x + Logback/Log4j2**，`@Slf4j` 完全兼容。

#### 五、面试必问：@Slf4j 与 传统 Logger 的区别

| 特性           | @Slf4j (Lombok)        | 传统手动 Logger                  |
| :------------- | :--------------------- | :------------------------------- |
| **代码量**     | **极少**，一个注解搞定 | **多**，每类都要写 static Logger |
| **运行时性能** | **一致**（编译时生成） | 一致                             |
| **可维护性**   | **高**，统一规范       | 低，容易写错类名                 |
| **IDEA 支持**  | **完美**（安装插件）   | 完美                             |
| **替代方案**   | —                      | 无                               |

**一句话总结**：
**@Slf4j 是 Lombok 提供的语法糖，它在编译期生成标准的 SLF4J Logger 代码，是现代 Java 后端开发的标准配置。**
